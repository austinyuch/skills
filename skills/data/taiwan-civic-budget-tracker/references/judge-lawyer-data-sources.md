# 法官與律師公開資料來源指南

## 概述

針對審判出庭律師（訴訟代理人）和法官的公開可查資料來源分析。

---

## 一、法官（Judge）公開資料來源

### 1. 司法院法官事務分配查詢

**資料來源**：司法院全球資訊網  
**URL**：https://www.judicial.gov.tw/tw/np-150-1.html

**可取得資訊**：
- 各級法院法官名單（最高法院、高等法院、地方法院）
- 法官事務分配（各法官負責的案件類型）
- 法院組織架構

**限制**：
- 僅顯示現任法官，不包含個人詳細資料（出生年、學經歷）
- 無搜尋功能，需逐法院查閱
- 資料更新頻率：不定期（通常隨人事異動更新）

**自動化難度**：★★★☆☆（中高）
- 網站有反爬蟲機制（驗證碼）
- 需處理不同法院的頁面結構差異

---

### 2. 司法院法學資料檢索系統（判決書查詢）

**資料來源**：司法院  
**URL**：https://lawsearch.judicial.gov.tw/

**可取得資訊**：
- **判決書中的法官資訊**：承審法官姓名（審判長、受命法官、陪席法官）
- 法官參與過的案件類型與數量（透過判決書統計）
- 法官的裁判風格與見解（透過判決內容分析）

**查詢方式**：
```
判決書欄位資訊：
- 臺灣臺北地方法院 105 年度訴字第 123 號民事判決
- 審判長法官 張某某
- 法官 李某某
- 法官 王某某
```

**限制**：
- 個資保護：判決書已隱藏當事人個資，但法官姓名公開
- 僅能查詢到參與過審判的法官，無完整法官名冊
- 最高法院、高等法院判決書較完整，地方法院部分簡易判決可能未上傳

**自動化難度**：★★☆☆☆（中低）
- 有查詢介面，但大量查詢會觸發反爬蟲
- 建議使用每日增量更新策略

---

### 3. 各法院開庭進度查詢系統

**資料來源**：司法院  
**URL**：https://csdi3.judicial.gov.tw/judbp/ctstate/index.jsp

**可取得資訊**：
- 特定案件的承審法官
- 開庭時間與進度

**限制**：
- 需知道案件案號才能查詢
- 僅顯示「現在進行式」的案件，已結案資訊會被移除
- 個資保護：僅限訴訟當事人查詢（需身分驗證）

**自動化難度**：★★★★☆（高）
- 需要登入驗證
- 僅能查詢特定案件，無法批量取得

---

### 4. 法官學院研究報告與出版品

**資料來源**：法官學院  
**URL**：https://tpi.judicial.gov.tw/

**可取得資訊**：
- 法官發表的學術文章、研究報告
- 進修與研討會記錄（可看出法官專業領域）

**限制**：
- 僅限參與研究或發表的法官
- 非強制性公開，資料不完整

---

### 5. 司法院資料開放平台

**資料來源**：司法院 Open Data  
**URL**：https://www.judicial.gov.tw/tw/cp-146-1.html

**可取得資訊**：
- 法院組織架構資料（JSON/XML格式）
- 部分統計數據

**限制**：
- 不包含個人層級的法官資料
- 多為結構化統計，非詳細名單

## 二、律師（Attorney）公開資料來源

### 1. 法務部律師查詢系統（推薦）

**資料來源**：法務部  
**URL**：https://lawyerbc.moj.gov.tw/

**可取得資訊**：
```json
{
  "律師姓名": "張某某",
  "性別": "男",
  "出生年份": "1975",
  "身分證字號": "A123456789", （僅顯示部分）
  "律師證書字號": "（71）台證字第1234號",
  "執業狀態": "正常",
  "所屬公會": "台北律師公會",
  "事務所名稱": "某某法律事務所",
  "事務所地址": "台北市...",
  "電話": "02-12345678",
  "電子郵件": "lawyer@example.com",
  "懲戒紀錄": [
    {
      "決議日期": "2020-05-15",
      "決議主文": "警告",
      "決議內文": "..."
    }
  ]
}
```

**特色功能**：
- ✅ **懲戒紀錄查詢**：可查詢律師是否曾被懲戒
- ✅ **執業狀態驗證**：確認是否為有效執業律師（防假冒）
- ✅ **事務所資訊**：聯絡方式、地址
- ✅ **出生年份**：用於年齡驗證與同名區分

**限制**：
- 更新頻率：每週五更新（非即時）
- 僅限執業律師，單純領有律師證書未執業者不提供查詢
- 不含律師專長領域、案件類型等資訊
- 需透過姓名、身分證字號或律師證書字號查詢

**自動化難度**：★★★☆☆（中等）
- 有網頁介面，可爬蟲
- 大量查詢可能觸發驗證碼
- 資料結構相對穩定

---

### 2. 全國律師聯合會（Taiwan Bar Association）

**資料來源**：全國律師聯合會  
**URL**：https://www.twba.org.tw/

**可取得資訊**：
- 全國執業律師統計數據（約13,000人）
- 律師公會活動資訊
- 全國執業制查詢（是否申請跨區執業）

**限制**：
- 不提供個別律師查詢功能
- 多為統計與活動資訊

---

### 3. 各地方律師公會

**資料來源**：17個地方律師公會  
**範例**：https://www.taipeibar.org.tw/（台北律師公會）

**可取得資訊**：
- 該地區會員律師名冊（部分公會公開）
- 律師專長領域分類（部分公會提供）
- 律師進修與活動紀錄

**限制**：
- 各公會公開程度不一
- 部分需登入會員才能查詢完整名冊
- 無統一查詢介面，需逐公會查詢

---

### 4. 判決書中的律師資訊

**資料來源**：司法院法學資料檢索系統  
**URL**：https://lawsearch.judicial.gov.tw/

**可取得資訊**：
- **訴訟代理人**：當事人委任的律師姓名
- **案件類型與結果**：律師代理的案件類型與判決結果
- **出庭紀錄**：律師參與過的庭期（從判決書日期推算）

**查詢範例**：
```
判決書內容：
「訴訟代理人 張某某律師 到庭」

可統計：
- 張某某律師代理過的案件類型分布
- 勝訴/敗訴比例（粗略統計）
- 常合作的當事人類型
```

**限制**：
- 判決書僅顯示當事人委任的律師，非所有出庭律師
- 民事案件通常有委任律師，刑事案件多由公設辯護人或無律師
- 無法區分「主辦律師」與「助理律師」

---

### 5. 全國律師事務所資料查詢

**資料來源**：全國律師聯合會  
**URL**：https://nab.twba.org.tw/lopen/pcinfo_add.aspx

**可取得資訊**：
- 合夥律師事務所名冊
- 事務所合夥人名單
- 事務所地址與聯絡方式

**限制**：
- 僅限「合夥」律師事務所
- 獨資與合署事務所不顯示
- 無個別律師查詢功能

## 三、資料來源整合建議

| 資料層級 | 來源 | 可用資訊 | 自動化可行性 |

## 四、技術實作注意事項

### 1. 個資保護紅線

**可以抓取**：
- ✅ 法官姓名（判決書公開資訊）
- ✅ 律師姓名、出生年、事務所（法務部公開資料）
- ✅ 懲戒紀錄（依法公開）
- ✅ 判決書內容（司法院公開資料）

**不可抓取**（可能有法律風險）：
- ❌ 法官完整個資（地址、電話、家庭成員）
- ❌ 律師完整個資（住家地址、私人電話）
- ❌ 未經公開的內部人事資料

### 2. 反爬蟲機制處理

司法院網站的防護措施：
- 驗證碼（CAPTCHA）保護
- IP 請求頻率限制
- Session 驗證

**建議策略**：
```python
# 1. 降低請求頻率
import time
time.sleep(2)  # 每請求間隔2秒以上

# 2. 使用 Proxy rotation
proxies = ['http://proxy1.com', 'http://proxy2.com']

# 3. 分散查詢時間
# 法官事務分配：每月查詢一次（異動不頻繁）
# 判決書：每日增量更新
# 律師資料：每週查詢一次（每週五更新）
```

### 3. 同名異人處理

**法官**：
```python
# 使用「法院+姓名」作為複合鍵
entity_id = f"{court_code}_{judge_name}"
# e.g., "TPD_張某某"（台北地院張某某）
```

**律師**：
```python
# 使用「姓名+出生年」作為複合鍵
entity_id = f"{lawyer_name}_{birth_year}"
# e.g., "張某某_1975"
```

## 五、實務應用場景

### 場景一：調查特定案件的法官與律師

```
目標：調查「某貪污案」的法官與辯護律師背景

執行步驟：
1. 判決書查詢系統 → 取得判決書
2. 提取法官姓名（審判長、受命法官）
3. 提取辯護律師姓名（訴訟代理人）
4. 法務部律師查詢 → 查詢律師背景、懲戒紀錄
5. 法官事務分配查詢 → 確認法官職務與專長領域
6. 新聞搜尋 → 查找法官與律師的公開發言
7. 交叉比對 → 是否有「旋轉門」關係（律師曾任法官）
```

### 場景二：建立「法律人」關係網絡

```
目標：建立法官-律師-當事人的關係網絡

執行步驟：
1. 批量下載判決書（特定法院、特定時期）
2. 提取所有法官與律師姓名
3. 建立關係：
   - 法官 → 審理過的案件
   - 律師 → 代理過的當事人
   - 律師 → 合作過的律師（同案件）
4. 分析：
   - 哪些律師常出現在特定法官的庭上？
   - 是否存在「常客」律師與法官的固定組合？
```

### 場景三：驗證法律人身分

```
目標：驗證「自稱律師」的人是否確實有執業資格

執行步驟：
1. 法務部律師查詢系統 → 輸入姓名
2. 檢查「執業狀態」是否為「正常」
3. 比對出生年份是否合理
4. 檢查是否有懲戒紀錄
5. 比對事務所名稱是否一致

警示：
- 查無此人 → 假冒律師
- 狀態「喪失資格」 → 已無執業資格
- 狀態「停止執行業務」 → 暫時不能執業
- 懲戒紀錄 → 注意違規歷史
```

## 六、結論與建議

### 法官資料可得性：**中等**

- ✅ **基礎名單**：可透過法官事務分配取得
- ✅ **案件關聯**：可透過判決書分析建立
- ❌ **詳細個資**：出生年、學經歷等較難取得
- ❌ **財產資訊**：需透過公職人員財產申報系統（限特定查詢）

### 律師資料可得性：**高**

- ✅ **身份驗證**：法務部系統提供完整執業資訊
- ✅ **懲戒紀錄**：依法公開，可查詢違規歷史
- ✅ **事務所資訊**：地址、聯絡方式公開
- ⚠️ **案件關聯**：需透過判決書分析，較間接

### 最佳實踐建議

1. **資料交叉驗證**
   - 判決書中的律師姓名 → 法務部系統驗證資格
   - 避免僅依賴單一來源

2. **隱私保護**
   - 僅收集公開職務相關資訊
   - 避免抓取私人聯絡資訊（住家電話、私人email）

3. **更新頻率**
   - 法官資料：每月更新（人事異動不頻繁）
   - 律師資料：每週更新（每週五法務部更新）
   - 判決書：每日增量更新

4. **實體管理**
   - 使用「姓名+出生年+所屬機構」作為複合識別
   - 建立「曾任法官轉任律師」的關係追蹤
   - 標記「常客」律師與法官的出庭組合
