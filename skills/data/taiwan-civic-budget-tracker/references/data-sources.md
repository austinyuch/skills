# 台灣政府預算與標案資料來源詳細說明

## 第一層：預算與補助資料

### 1. 主計總處預算視覺化

**來源**：主計總處歲計會計資訊網
**URL**：https://www.dgbas.gov.tw

**資料內容**：
- 總預算歲出計畫
- 部會預算分類（依政務分類、經濟分類）
- 預算執行進度

**存取方式**：
- 網頁瀏覽（https://budget.dgbas.gov.tw）
- CSV/Excel 下載（年度預算書）
- API：無正式 API，需爬蟲

**更新頻率**：年度（每年 8-9 月公布下年度預算）

**自動化難度**：★★★☆☆（中等）
- 網站有反爬蟲機制
- 需要處理 JavaScript 渲染
- 建議使用 Playwright 或 Selenium

### 2. 公職人員利益衝突迴避法平台

**來源**：財政部廉政署
**URL**：https://published-info.mof.gov.tw

**資料內容**：
- 公職人員財產申報
- 公職人員及關係人補助交易
- 公職人員及關係人買賣有價證券
- 公職人員及關係人捐贈資訊

**關鍵欄位**：
- 公職人員姓名、服務機關
- 補助對象（關係人姓名/公司名稱）
- 補助金額、補助期間
- 補助機關

**存取方式**：
- 網頁搜尋介面
- 無公開 API
- 需使用爬蟲（有驗證碼保護）

**自動化難度**：★★★★☆（高）
- 有驗證碼（CAPTCHA）
- 搜尋結果分頁顯示
- 建議使用 Playwright + 打碼服務

**策略建議**：
1. 使用 Playwright 自動化瀏覽器
2. 對於驗證碼，可考慮 2Captcha 等服務
3. 建立搜尋關鍵字清單，分批查詢
4. 使用 Proxy rotation 避免被封鎖

## 第二層：採購與標案資料

### 4. 政府電子採購網（PCC）

**來源**：行政院公共工程委員會
**URL**：https://web.pcc.gov.tw

**資料類型**：

#### 4.1 公開資料查詢
**URL**：https://web.pcc.gov.tw/pis/

**可查詢資料**：
- 招標公告（尚未決標）
- 決標公告（已完成）
- 公開閱覽資料
- 無法決標公告

**關鍵欄位**：
```
標案案號、標案名稱
招標機關、招標方式
決標金額、決標日期
得標廠商統編、廠商名稱
投標廠商數量、履約期限
```

#### 4.2 公開資料下載
**URL**：https://web.pcc.gov.tw/pis/openData/

**開放資料格式**：
- CSV（招標公告、決標公告）
- XML（採購資料）
- 更新頻率：每日

**存取方式**：
```python
import pandas as pd

# 直接下載 CSV
url = "https://web.pcc.gov.tw/pis/openData/awardCSV"
df = pd.read_csv(url, encoding='utf-8')
```

#### 4.3 標案資料查詢系統（需要帳號）
- 更詳細的廠商資料
- 歷史標案紀錄
- 需要申請帳號

**自動化難度**：
- 公開 CSV：★☆☆☆☆（極低）
- 網頁搜尋：★★★☆☆（中等）
- 登入系統：★★★★☆（高）

**注意事項**：
1. 公開 CSV 為每日更新，非即時
2. 網頁搜尋有查詢次數限制
3. 大量爬取可能被封鎖 IP
4. 建議使用公開資料為主，網頁爬蟲為輔

## 資料整合策略

### 推薦優先順序

1. **標案資料**：優先使用 g0v PCC API，其次官方 PCC CSV
2. **公司資料**：優先使用 twcom，其次經濟部開放資料
3. **預算資料**：手動下載 + PDF 解析（自動化難度高）
4. **補助資料**：需客製化爬蟲（有驗證碼）

### 資料更新策略

| 資料類型 | 更新頻率 | 建議策略 |
|---------|---------|---------|
| 標案決標 | 每日 | 自動化排程，每日抓取 |
| 公司資料 | 每月 | 批次更新，增量同步 |
| 預算資料 | 年度 | 年度手動匯入 |
| 補助資料 | 即時 | 依調查需求手動查詢 |

### 資料品質注意事項

1. **時間延遲**：
   - 標案從招標到決標可能數月
   - 商工登記變更有時間差

2. **資料缺漏**：
   - 部分標案未公開完整資訊
   - 小公司董監事資料可能不完整

3. **格式不一致**：
   - 機關名稱簡寫（如「法務部」vs「法務部調查局」）
   - 公司名稱後綴（「股份有限公司」有無）
   - 金額單位（元、萬元、千元）

4. **同名異人**：
   - 人名比對需配合其他資訊（公司、時間）
   - 建議建立人員唯一識別機制
