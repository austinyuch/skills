# 第三階段：任務規劃

## 觸發條件

- 設計文件已完成並獲得您的認可
- 或您表示要規劃實施任務

## 我的工作

1. 調整到 ultrawork 模式
2. 必須完全了解專案程式碼，了解專案規範和具體實現，收集足夠完整的上下文，再開始工作
3. 仔細研讀需求和設計文件
4. 在 active root 下創建 `{.agents, .kiro, or .claude}/specs/{功能名}/tasks.md` 文件，文件開頭必須說明相關章節的引用參照；本專案目前建立於 `.agents/specs/{功能名}/`
5. 將功能設計轉換為一系列程式碼生成 LLM 的提示，以測試驅動的方式實現每個步驟
6. 優先考慮最佳實踐、漸進式進展和早期測試，確保任何階段都不會有複雜度的大跳躍
7. 確保每個提示都建立在之前的提示基礎上，最後將所有內容連接起來
8. 不應有懸空或孤立的程式碼未整合到之前步驟中
9. **只專注於涉及編寫、修改或測試程式碼的任務**

## 任務格式要求

- 使用帶編號的核取方塊列表，最多 3 級層次結構；例如：`- [ ] {任務編號} {任務標題}`
- 頂層項目（如史詩）僅在需要時使用
- 子任務使用小數記號編號（例如 1.1、1.2、2.1）
- 每項必須是核取方塊
- 優先選擇簡單結構

## 每個任務項必須包含

- 涉及編寫、修改或測試程式碼的清晰目標作為任務描述
- 任務下的子項附加資訊
- 對需求文件（requirements.md）中要求的具體引用（引用細粒度子需求，而非僅用戶故事）

## 任務內容約束

- 必須是一系列離散、可管理的編碼步驟
- 每個任務引用需求文件中的特定需求，並標註是滿足哪些 requirements，以利需求追溯（traceability）
- 不包含設計文件中已涵蓋的過度實現細節
- 假設所有上下文文件（功能需求、設計）在實現期間都可用
- 每個步驟在之前步驟的基礎上漸進式構建
- 應優先考慮適當的測試驅動開發（TDD workflow）
- 覆蓋設計中可通過程式碼實現的所有方面
- 應排序步驟以通過程式碼早期驗證核心功能
- 確保所有需求都被實現任務覆蓋
- 預設不包含測試部分的任務，除非用戶特別說明
- 設計時考量平衡 spec 內分 branch 平行開發的可能性，但儘量減少整合複雜度，並設計「移除 mock 採用真實實作程式並且重新測試」的整合任務
- 需求任務變更時，檢查 tasks.md 行數，如果超過 3000 行，請將已經完成的逐 phase 備份到 `tasks-{phase-number-and-alias}.md`，並且在 tasks.md 該 phase 建立摘要和引用該 archived phase 文件路徑，以利未來追溯

## 任務必須是程式碼代理可執行的

- 任務應涉及編寫、修改或測試特定程式碼元件
- 任務應指定需要創建或修改的文件或元件
- 任務應具體到程式碼代理可以執行而無需額外澄清
- 任務應專注於實現細節而非高層概念
- 任務應限定為特定編碼活動（如「實作 X 函式」而非「支援 X 功能」）

## 明確避免的非編碼任務

- 用戶驗收測試或用戶反饋收集
- 部署到生產或預備環境
- 效能指標收集或分析
- 執行應用程式端到端測試（但可以編寫自動化測試從用戶角度測試端到端）
- 用戶培訓或文件創建
- 業務流程變更或組織變更
- 行銷或溝通活動
- 任何無法通過編寫、修改或測試程式碼完成的任務

## 示範格式

```markdown
# 實施計畫

- [ ] 1. 設置專案結構和核心介面
  - 為模型、服務、儲存庫和 API 元件創建目錄結構
  - 定義建立系統邊界的介面
  - _需求: 1.1_

- [ ] 2. 實作資料模型和驗證
- [ ] 2.1 建立核心資料模型介面和型別
  - 為所有資料模型編寫 TypeScript 介面
  - 實作資料完整性驗證函式
  - _需求: 2.1, 3.3, 1.2_

- [ ] 2.2 實作帶驗證的 User 模型
  - 編寫帶驗證方法的 User 類別
  - 為 User 模型驗證創建單元測試
  - _需求: 1.2_

[其他編碼任務繼續...]
```

## 自然地詢問

「任務規劃看起來合理嗎？如果同意的話，我們可以開始逐個實作了」

## 重要約束

- 如果用戶表示需要對設計進行任何更改，必須返回設計步驟
- 如果用戶表示需要額外需求，必須返回需求步驟
- 必須等待您的明確認可才能完成工作流程
- 如果您要求更改或未明確批准，我必須修改任務文件
- 每次編輯後必須明確請求批准
- 必須持續「反饋－修訂」循環直到獲得明確批准
- 如果在實施規劃期間發現差距，會建議返回之前步驟（需求或設計）

## 完成標誌

您明確同意任務規劃
