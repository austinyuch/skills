# Vibe Skill 快速開發工作流

這是一個快速開發的 OpenCode Skill，幫助您快速、高效地完成開發任務，同時保持程式碼品質。

## Skill 結構

```
vibe-skill/
├── SKILL.md                 # 主檔案：核心工作流程和全域約束
├── README.md                # 本檔案：使用說明
└── phases/                  # 階段指南檔案夹
    ├── 01-understand.md     # 第一階段：理解
    ├── 02-plan.md           # 第二階段：規劃
    └── 03-execute.md        # 第三階段：執行
```

## 何時使用這個 Skill

當你需要：

- 快速實作小功能
- 修復 Bug
- 程式碼重构
- 快速原型驗證
- 小型迭代開發

Agent 会自動识別並使用這個 Skill。

## 三階段工作流

```
理解 (Understand) → 規劃 (Plan) → 執行 (Execute)
```

1. **理解** - 深入理解項目現状、程式碼規範和使用者需求
2. **規劃** - 與使用者沟通方案，建立详尽的任務清單格
3. **執行** - 按計畫快速、高品質地完成任務

## 與六階段工作流的對比

| 場景             | 推荐工作流                  |
| ---------------- | --------------------------- |
| 複雜新功能開發   | kiro-laravel-skill (六階段) |
| 快速原型驗證     | vibe-skill (三階段)         |
| 小功能迭代       | vibe-skill (三階段)         |
| Bug 修復         | vibe-skill (三階段)         |
| 程式碼重构       | vibe-skill (三階段)         |
| 需要嚴格品質控制 | kiro-laravel-skill (六階段) |

## 全域約束

- 使用 ultrathink 模式深度思考
- 必須完全瞭解項目程式碼和規範
- 語言：繁體中文
- 嚴格遵守 `.kiro/steering` 目錄規則
- 執行任務前建立 To-dos，精确到每個子任務
- 不自動執行 SQL 語句
- 從全域到細節的方法論
- 按業務模組開發，不按程式碼分層

## 使用示例

### 開始新任務

```
使用者：幫我實作一個商品庫存预警功能
Claude：好的，讓我先理解一下項目現状...
       [理解階段]
       基於你的需求，我建議這樣實作...
       [規劃階段，建立 todos]
       任務清單格已经建好了，確認後我開始執行？
```

### 快速修復

```
使用者：订單清單格介面返回資料格式不對
Claude：讓我先看看相关程式碼...
       [快速定位問題]
       找到問題了，我来修復它...
       [建立簡單 todos，快速修復]
```

## 核心特點

1. **充分理解** - 開始前深入理解項目和需求
2. **提前沟通** - 規劃階段和使用者充分沟通方案
3. **任務透明** - 详尽的 todos 讓使用者瞭解進度
4. **規範优先** - 嚴格遵守項目程式碼規範
5. **快速迭代** - 保持高效但不牺牲品質
6. **主動完成** - 主動推進工作，减少使用者推動

## 階段檔案說明

### phases/ 目錄

包含三個階段的詳細實施指南：

- **01-understand.md** - 詳細說明如何理解項目和需求，包括規範收集、程式碼分析
- **02-plan.md** - 詳細說明如何與使用者沟通方案，如何建立详尽的任務清單格
- **03-execute.md** - 詳細說明如何高效執行任務，包括任務流程、程式碼規範、進度更新

## 注意事項

1. **理解階段不能跳过** - 必須先理解項目規範
2. **規劃階段需要使用者確認** - 方案和任務清單格需要使用者认可
3. **任務編號必須清晰** - 格式為"編號 + 空格 + 任務描述"
4. **實時更新進度** - 讓使用者始終瞭解當前狀態
5. **SQL 脚本需要使用者手動執行**

## 工作風格

Vibe 采用轻松、友好的互動方式：

- 用自然語言沟通，像和同事交流
- 提供解決方案导向的建議
- 主動完成工作，减少使用者推動
- 保持積极乐观的態度

透過這種快速、高效的協作方式，確保在保持程式碼品質的同時大幅提升開發效率。
