# PRD評審標準

本文件定義PRD需求評審的評分標準和品質評級机制，確保PRD文件品質的客观評估。

## 評審概述

### 評審目的

- 客观評估PRD文件品質
- 识别需求文件中的問題和风險
- 提供具體的改进建議
- 確保PRD具备可執行性

### 評審原則

- **客观公正**：基於事实和標準，避免主观臆断
- **全面系統**：覆盖需求、設計、業務、文件等多個維度
- **具體可操作**：問題描述清晰，建議切实可行
- **持续改进**：支援多轮優化，逐步提升品質

## 評估維度

PRD評審采用五維度評估体系，总分10分：

| 維度 | 权重 | 说明 |
|------|------|------|
| 需求完整性 | 25% | 功能覆盖度、边界條件、异常處理 |
| 可執行性 | 25% | 開發理解度、資源匹配度、驗收標準 |
| 使用者價值 | 20% | 需求匹配度、體驗設計、價值创造 |
| 業務邏輯 | 20% | 流程合理性、規則完整性、資料一致性 |
| 文件品質 | 10% | 結構清晰度、描述准确性、格式規範性 |

## 詳細評分標準

### 1. 需求完整性（25%，满分2.5分）

#### 1.1 功能覆盖度（10%，满分1.0分）

**評分標準**：

| 得分区间 | 評估標準 |
|----------|----------|
| 0.9-1.0 | 所有核心功能完整覆盖，主要業務場景全部描述，使用者角色和權限定義清晰 |
| 0.7-0.8 | 核心功能基本覆盖，主要場景有描述，角色權限基本明确 |
| 0.5-0.6 | 核心功能部分遗漏，場景描述不够全面，角色權限定義模糊 |
| 0.3-0.4 | 功能覆盖度较低，重要場景缺失，角色權限未明确 |
| 0.0-0.2 | 功能严重缺失，場景描述不清，无角色權限定義 |

**檢查要点**：
- [ ] 產品規劃中的所有功能都有詳細描述
- [ ] 核心業務場景全部覆盖
- [ ] 所有使用者角色的權限都已定義
- [ ] 功能边界清晰明确

#### 1.2 边界條件（8%，满分0.8分）

**評分標準**：

| 得分区间 | 評估標準 |
|----------|----------|
| 0.7-0.8 | 明確定義功能边界，清晰说明什么不做，約束條件完整 |
| 0.5-0.6 | 边界基本明确，約束條件基本完整 |
| 0.3-0.4 | 边界定義模糊，部分約束條件缺失 |
| 0.0-0.2 | 边界不清晰，約束條件严重缺失 |

**檢查要点**：
- [ ] 明确说明功能范围和边界
- [ ] 清晰定義什么在范围内，什么不在范围内
- [ ] 说明技術約束、資源約束、时间約束
- [ ] 定義与其他系統的集成边界

#### 1.3 异常處理（7%，满分0.7分）

**評分標準**：

| 得分区间 | 評估標準 |
|----------|----------|
| 0.6-0.7 | 异常場景全面考虑，错误處理規則明确，容错机制完善 |

## 品質評級

### 評分計算

综合得分 = 需求完整性得分 + 可執行性得分 + 使用者價值得分 + 業務邏輯得分 + 文件品質得分

### 品質分級

| 等級 | 得分范围 | 品質評价 | 建議行動 |
|------|----------|----------|----------|
| 优秀 | ≥8.5 | PRD品質优秀，可直接發布 | 进入發布准备階段 |
| 良好 | 7.0-8.4 | PRD品質良好，建議優化后發布 | 进入文件優化階段 |
| 需要優化 | 6.0-6.9 | PRD存在明显問題，需要優化 | 进入文件優化階段，重点修复主要問題 |
| 必須優化 | <6.0 | PRD品質不达标，必須大幅改进 | 进入文件優化階段，全面改进 |

### 評級詳細说明

#### 优秀（≥8.5分）

**特征**：
- 需求完整，覆盖全面
- 描述清晰，无歧义
- 業務邏輯合理完整
- 使用者價值明确
- 文件品質高

**建議**：
- 可以直接进入發布准备
- 制作最終交付版本
- 分发给開發团队

#### 良好（7.0-8.4分）

**特征**：
- 需求基本完整
- 描述基本清晰
- 業務邏輯基本合理
- 有明确的使用者價值
- 文件品質良好

**建議**：
- 建議进行一轮優化
- 修复主要問題
- 提升到8.5分以上

#### 需要優化（6.0-6.9分）

**特征**：
- 需求不够完整
- 部分描述不清晰
- 業務邏輯有不合理之处
- 使用者價值不够突出
- 文件品質一般

**建議**：
- 必須进行優化
- 重点修复P0和P1問題
- 目标提升到7.5分以上

#### 必須優化（<6.0分）

**特征**：
- 需求缺失较多
- 描述模糊或有歧义
- 業務邏輯存在严重問題
- 使用者價值不明确
- 文件品質较差

**建議**：
- 必須大幅改进
- 全面修复各类問題
- 可能需要重新梳理需求和規劃

## 評審報告模板

評審報告應包含以下內容：

1. **評審概要**：評審时间、版本、得分、结论
2. **評分详情**：五個維度的詳細評分和说明
3. **综合評估**：主要优点、主要問題、優化建議
4. **評審结论**：品質評級和下一步建議

詳細模板请參考 [phases/04-review.md](../phases/04-review.md)

## 評審流程

1. **准备評審**：确认PRD文件草稿已完成
2. **逐項評估**：按照評分標準逐項評估
3. **計算得分**：匯總各項得分得出综合得分
4. **编写報告**：撰寫詳細的評審報告
5. **反馈沟通**：与產品經理沟通評審结果
6. **確定后续**：决定是否进入優化階段

## 注意事項

1. **客观評審**：基於標準和事实，避免主观判断
2. **全面分析**：不遗漏任何重要問題
3. **具體建議**：提供可操作的改进方案
4. **正面激励**：肯定优点，鼓励改进
5. **持续跟踪**：跟踪優化效果，確保品質提升
