# Tag 註冊表與管理系統

本文檔說明如何使用 Tag 系統追蹤特定事件和主題的連續新聞報導。

## Tag 系統概述

Tag 是受害者家屬權益新聞追蹤器的核心功能，用於：

1. **標記特定案件**：如 `#普悠瑪事故`、`#鄭捷案`
2. **標記主題類別**：如 `#司法不公`、`#人權活動`
3. **標記追蹤狀態**：如新聞的重要程度、後續行動需求
4. **建立事件關聯**：連結相關的新聞報導形成時間線

## Tag 命名規範

### 命名原則

1. **簡潔明確**：2-6 個字為佳，避免過長
2. **無空格**：使用中文或英文，不含空格
3. **前綴符號**：使用 `#` 作為 tag 識別符號
4. **避免特殊字元**：不使用標點符號（破折號、引號等）

### 命名範例

**案件類 Tag**：
- ✅ `#普悠瑪事故`
- ✅ `#鄭捷案`
- ✅ `#小燈泡案`
- ❌ `#普悠瑪 事故`（含空格）
- ❌ `#普悠瑪號事故`（過長）

**主題類 Tag**：
- ✅ `#司法不公`
- ✅ `#家屬權益`
- ✅ `#人權活動`
- ❌ `#司法 不公`（含空格）

**狀態類 Tag**：
- ✅ `#進行中`
- ✅ `#待開庭`
- ✅ `#持續關注`

### 建議的標準化 Tag

#### 主題分類標籤（必用）

```
#司法不公      - 突顯司法偏向加害者的報導
#家屬權益      - 關於受害者家屬權利爭取的報導
#人權活動      - 人權團體為受害者發聲的活動
#案件追蹤      - 特定案件的連續報導
#國際案例      - 國際間相關案例（非台灣）
#修法倡議      - 關於法律改革的倡議報導
#賠償爭議      - 關於民事賠償的爭議報導
#立法倡議      - 法案推動、修法進展
#司法改善      - 制度改革、政策倡議、司改進展
```

#### 立法司法改善專屬標籤（選用）

```
#法案進展      - 具體法案的審查進度（一讀、二讀、三讀）
#司改進展      - 司法改革相關制度建立或修正
#制度建立      - 新的保護制度或機制建立
#修法通過      - 法案已完成修法程序
#修法審查中    - 法案正在立法院審查階段
#政策倡議      - 政府或團體提出的政策方向
#國際公約      - 與國際人權公約落實相關
```

#### 狀態標籤（選用）

```
#進行中        - 案件正在司法程序中
#待開庭        - 即將開庭審理
#上訴中        - 正在上訴階段
#已判決        - 已有判決結果
#已結案        - 案件已完全結束
#持續關注      - 需要長期追蹤的議題
```

#### 重要性標籤（選用）

```
#重大案件      - 社會矚目的重大刑事案件
#社會關注      - 引發廣泛社會討論的案件
#人權侵害      - 涉及人權侵害的嚴重案件
#制度缺陷      - 突顯制度問題的案件
```

#### 角色立場標籤（選用 - 用於人物圖像建立）

```
#NPO發聲       - 非營利組織發言、聲明、活動
#政黨立場      - 政黨、政治人物相關發言與立場
#法界觀點      - 法官、律師等法律人士的評論與見解
```

#### 角色類型標籤（選用 - 以 @ 符號標記具體人物/組織）

```
@組織          - 非營利組織（如 @生命權平等協會）
@政黨          - 政黨官方（如 @民進黨 @國民黨）
@立委          - 立法委員（如 @王婉諭 @黃國昌）
@法官          - 法官
@檢察官        - 檢察官
@律師          - 律師
@官員          - 政府官員（如法務部長、司法院長等）
```

#### 立場傾向標記系統（⭐ 星級標記）

使用星級系統標記角色在受害者家屬權益議題上的立場傾向：

| 標記 | 名稱 | 定義 | 判斷標準 |
|------|------|------|----------|
| ⭐⭐⭐⭐⭐ | 強烈支持 | 積極推動修法、多次為家屬發聲、具體實質行動 | 有實際提案、參與記者會、持續追蹤案件 |
| ⭐⭐⭐⭐ | 支持 | 公開表態支持、參與相關活動 | 發表聲明、出席活動、表示支持立場 |
| ⭐⭐⭐ | 中立 | 表示關注但未明確表態，或提出平衡觀點 | 僅表示關注、未明確支持或反對 |
| ⭐⭐ | 消極 | 態度模糊、迴避表態、僅官方客套話 | 官方回應、無實質內容、迴避核心議題 |
| ⭐ | 反對/阻礙 | 明確反對家屬訴求、阻撓法案進度 | 公開反對、阻擋修法、負面評價 |

#### 關係類型標籤（選用）

```
#同盟          - 與其他角色共同支持家屬權益
#對立          - 與其他角色持相反立場
#聯合聲明      - 與其他角色共同發表聲明
#公開辯論      - 與其他角色有公開意見交鋒
#上下級        - 組織內的上下級關係（如政黨-立委）
```

```markdown

```

## Tag 管理流程

### 註冊新 Tag

當需要追蹤新的特定事件或主題時：

1. **評估必要性**
   - 是否為重要案件或主題？
   - 是否會有連續報導？
   - 是否值得長期追蹤？

2. **選擇適當名稱**
   - 參考「命名規範」
   - 檢查是否與現有 Tag 重複
   - 確保名稱簡潔明確

3. **建立完整記錄**
   - 填寫所有必要欄位
   - 記錄相關關鍵詞供後續搜尋使用
   - 設定初始追蹤狀態

4. **執行首次搜尋**
   - 使用相關關鍵詞搜尋歷史報導
   - 為找到的新聞標記新 Tag
   - 更新「相關新聞數量」

### 更新 Tag 狀態

每次執行每日更新時：

1. **檢查所有進行中/待開庭/上訴中 Tag**
2. **搜尋該 Tag 的最新報導**（限制時間為過去 24 小時）
3. **更新狀態**：如有重大進展，更新「追蹤狀態」
4. **記錄關鍵進展**：簡要描述最新情況
5. **更新最後檢查日期**
6. **累計新聞數量**

### 結案或暫停追蹤

當案件結束或不再活躍時：

1. **更新狀態為「已結案」或「持續關注」**
2. **記錄最終結果**：簡要說明案件最終狀態
3. **降低檢查頻率**：已結案 Tag 可改為每週檢查一次

### Tag 清理

定期（建議每季度）檢查：

1. **移除未使用的 Tag**：長期無相關新聞的 Tag
2. **合併重複 Tag**：如發現相似的案件或主題
3. **更新過期資訊**：修正錯誤的案件背景或資訊

### 角色識別與記錄流程

**Step 1：在新聞中識別角色**
- 閱讀新聞時注意提到的組織、個人
- 標記發言者的角色類型

**Step 2：判斷是否需要建立檔案**
建立檔案的條件：
- 首次發現該角色發言
- 該角色具有影響力或代表性
- 該角色的立場具有記錄價值

**Step 3：擷取關鍵資訊**
- 直接引用或摘要其發言
- 記錄發言場合與情境
- 初步判斷立場傾向

**Step 4：建立或更新檔案**
- 新角色：建立新檔案，填寫完整資訊
- 已知角色：在現有檔案中新增發言記錄

**Step 5：標記關聯關係**
- 識別與其他角色的互動（同盟、對立等）
- 更新人物關係圖

### 立場判斷準則

**⭐⭐⭐⭐⭐ 強烈支持（積極行動派）**
- 特徵：
  - 主動提案修法或政策
  - 多次參與記者會、活動
  - 持續追蹤案件進展並發聲
  - 實際協助家屬維權行動
- 範例：提案修法立委、長期倡議NPO、義務代理律師

**⭐⭐⭐⭐ 支持（明確表態派）**
- 特徵：
  - 公開發表支持聲明
  - 出席相關活動表示支持
  - 在質詢或訪談中明確支持立場
- 範例：表示支持的政黨發言人、表態支持修法的地方首長

**⭐⭐⭐ 中立（觀望平衡派）**
- 特徵：
  - 表示關注但未明確支持
  - 提出平衡觀點（既提問題也提解方）
  - 未明確表態支持或反對
- 範例：僅表示「將研議」的官員、提出技術性問題的專家

**⭐⭐ 消極（迴避模糊派）**
- 特徵：
  - 官方制式回應，無實質內容
  - 明確迴避表態
  - 僅客套性關懷，無具體承諾
- 範例：「尊重司法獨立」的官方回應、僅表示「遺憾」的發言

**⭐ 反對/阻礙（負面行動派）**
- 特徵：
  - 公開反對修法或家屬訴求
  - 在程序上阻撓法案進度
  - 對家屬權益持負面評價
- 範例：明確反對修法的立委、做出輕判且理由引發爭議的法官

## 每日更新檢查清單

執行每日更新時，對每個追蹤中 Tag 檢查：

- [ ] 搜尋該 Tag 相關的新聞（過去 24 小時）
- [ ] 是否有新報導？記錄數量和內容摘要
- [ ] 是否有重大進展？更新「關鍵進展」欄位
- [ ] 追蹤狀態是否需要變更？（如從「進行中」改為「已判決」）
- [ ] 更新「最後更新」日期
- [ ] 累計「相關新聞數量」

## Tag 使用技巧

### 多 Tag 組合

一篇新聞可以標記多個 Tag，例如：
- 小燈泡案的最新報導：`#小燈泡案` `#案件追蹤` `#司法不公`
- 人權團體活動報導：`#人權活動` `#生命權平等協會` `#家屬權益`

### Tag 層級結構

建議建立階層式 Tag 系統：


### 交叉引用

在 Tag 記錄中引用相關 Tag：

```markdown
### #普悠瑪事故

**相關 Tag**：#司法不公 #國家賠償 #鐵路安全  
**常與以下 Tag 並用**：#家屬權益 #案件追蹤
```

## 範例：從註冊到追蹤的完整流程

### 情境：發現新的重大案件需要追蹤

**第 1 天：註冊 Tag**

1. 使用者提供資訊：「請追蹤某某命案的後續發展」
2. 註冊新 Tag：`#某某命案`
3. 填寫完整記錄：
   - 案件背景、相關關鍵詞、追蹤狀態
4. 執行首次搜尋：
   - 搜尋案件名稱 + 家屬
   - 找到 5 篇相關報導
   - 為每篇標記 `#某某命案` `#案件追蹤` 等 Tag

**第 2-N 天：每日追蹤**

1. 每日搜尋：`#某某命案` 相關新聞（過去 24 小時）
2. 發現新報導：新增 1 篇報導
3. 標記新報導：`#某某命案` `#新報導`
4. 更新 Tag 狀態：
   - 更新「最後更新」日期
   - 累計「相關新聞數量」為 6 篇
   - 如有重大進展，更新「關鍵進展」

**結案階段**

1. 案件判決已定讞
2. 更新追蹤狀態為「已結案」
3. 記錄最終結果
4. 改為每週檢查一次

---

**提示**：初次使用此系統時，建議先建立 5-10 個最重要的案件 Tag，並熟悉追蹤流程後再逐步擴充。
