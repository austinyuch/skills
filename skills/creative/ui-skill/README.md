# UI Skill - B端供應鏈靜態互動HTML產生器

專業的B端供應鏈靜態互動原型產生助手，幫助團隊快速將產品需求轉化為可互動的HTML原型。

## 快速開始

### 基本使用

直接在OpenCode中自然語言互動即可，無需記憶命令：

```
"我需要產生一個供應商清單格頁面"
```

或

```
"這是我們的PRD檔案（附件），幫我產生靜態互動原型"
```

Claude会自動调用UI Skill，透過六個系統化的階段幫您產生高品質的HTML原型。

## 核心功能

### 1. 快速原型產生

將PRD檔案或功能描述快速轉化為可互動的HTML原型

### 2. 多模板支援

- **AT-UI風格**（預設）：現代、動感、丰富
- **Ant Design風格**：專業、簡洁、高效
- **Element Plus風格**：清新、友好、易用

### 3. 完整互動模擬

- 完整的CRUD操作
- 表格單驗證
- 狀態管理
- 模擬資料

### 4. B端供應鏈專業性

深度理解採購、庫存、物流、供應商管理等核心業務場景

### 5. 開發友好

產生的程式碼規範清晰，可直接作為開發參考或基础

## 工作流程

UI Skill采用六階段系統化工作流：

```
1. 需求理解 → 2. 介面規劃 → 3. 模板選擇
     ↓              ↓              ↓
4. 程式碼產生 → 5. 互動驗證 → 6. 最佳化交付
```

### 階段一：需求理解

- 识別輸入類型（功能描述或PRD檔案）
- 提取核心功能模組和頁面類型
- 互動式確認需求理解
- 產生需求理解檔案

### 階段二：介面規劃

- 設計頁面佈局和回應式方案
- 規劃功能模組
- 設計互動流程
- 定義資料結構
- 產生介面規劃檔案

### 階段三：模板選擇

- 展示可用模板
- 推荐最合適的模板
- 使用者確認選擇
- 載入模板資源
- 產生模板選擇記錄

### 階段四：程式碼產生

- 逐頁面產生HTML程式碼
- 實作所有規劃的功能
- 添加詳細程式碼註解
- 確保程式碼可直接執行
- 產生使用說明檔案

### 階段五：互動驗證

- 驗證所有功能實作
- 測試互動功能
- 檢查程式碼品質
- 測試浏覽器相容性
- 產生驗證報告

### 階段六：最佳化交付

- 根据驗證反馈最佳化
- 完善檔案
- 最終交付

## 常见問題

### Q: 如何開始使用UI Skill？

A: 直接在OpenCode中用自然語言描述您的需求即可：

- "產生一個採購订單清單格頁面"
- "這是PRD檔案，幫我產生互動原型"

Claude会自動调用UI Skill並引導您完成整個流程。

### Q: 可以同時產生多個頁面吗？

A: 可以。UI Skill会根据您的需求自動規劃需要產生几個頁面，並逐個產生。

### Q: 產生的HTML可以直接用於生產環境吗？

A: 不建議。UI Skill產生的是靜態互動原型，主要用於：

- 需求驗證和演示
- 團隊沟通
- 開發參考

如需生產環境應用，需要進一步開發並對接真實後端API。

### Q: 可以修改產生的HTML吗？

A: 當然可以。產生的HTML程式碼結構清晰、註解完善，易於理解和修改。

### Q: 如何添加自定義模板？

A: 在 `templates/custom/` 目錄下建立新檔案夹，添加 `template.html` 和 `README.md` 即可。

### Q: 產生過程中可以中断吗？

A: 可以。UI Skill支援在任何階段暂停，下次说"繼續上次產生"即可繼續。

### Q: 可以返回修改前面的階段吗？

A: 可以。在任何階段都可以要求返回修改前面的內容。
