# 決策矩陣：何時使用 Rust 優化

## 資料大小閾值

| 資料大小 | 建議 | 原因 |
|---------|------|------|
| < 1KB | 保留 Go | CGO 呼叫開銷 > 效能增益 |
| 1KB - 100KB | 視情況 | 需要 benchmark 驗證 |
| > 100KB | 使用 Rust | 計算密集，Rust 優勢明顯 |

## 運算類型評估

### 高優先級 (強烈建議 Rust)

- ✅ 矩陣運算 (線性代數)
- ✅ 加密/解密 (AES, RSA)
- ✅ 影像處理 (濾鏡, 轉換)
- ✅ 信號處理 (FFT, DSP)
- ✅ 壓縮/解壓縮
- ✅ 自定義 Binary Protocol 解析

### 中優先級 (需要 benchmark)

- ⚠️ 排序演算法 (大資料集)
- ⚠️ 字串處理 (大量正則表達式)
- ⚠️ JSON/XML 解析 (複雜結構)
- ⚠️ 資料庫查詢結果處理

### 低優先級 (保留 Go)

- ❌ 簡單的 CRUD 操作
- ❌ HTTP 請求處理
- ❌ 業務邏輯判斷
- ❌ 小型資料結構操作

## GC 壓力評估

### 高 GC 壓力 (建議 Rust)

```go
// Pattern: 迴圈內大量小物件分配
for i := 0; i < 1000000; i++ {
    obj := &MyStruct{...}  // 每次迴圈分配新物件
    result = append(result, obj)
}
```

### 中 GC 壓力 (視情況)

```go
// Pattern: 預分配但有 interface{} 轉換
data := make([]interface{}, 0, 1000)
for _, item := range items {
    data = append(data, item)  // Boxing
}
```

### 低 GC 壓力 (保留 Go)

```go
// Pattern: 預分配且無額外分配
result := make([]int, len(input))
for i, v := range input {
    result[i] = v * 2
}
```

## CPU 使用率閾值

根據 `pprof` 分析結果：

| CPU 使用率 | 建議 | 行動 |
|-----------|------|------|
| > 15% | 強烈建議 | 立即轉換為 Rust |
| 10-15% | 建議 | Benchmark 後決定 |
| 5-10% | 可選 | 低優先級優化 |
| < 5% | 不建議 | 保留 Go |

## 效能增益預期

### 預期加速比

| 場景 | 預期加速 | 實際案例 |
|------|---------|---------|
| 數學密集運算 | 3-5x | 矩陣乘法: 4.2x |
| SIMD 可用運算 | 4-8x | 向量運算: 6.5x |
| 記憶體密集操作 | 2-3x | 大陣列處理: 2.8x |
| 混合運算 | 1.5-2.5x | 一般演算法: 2.1x |

### 最小效能增益要求

- **開發環境**: 至少 1.5x 加速才值得轉換
- **生產環境**: 至少 2x 加速才值得維護成本
- **嵌入式環境**: 至少 3x 加速 (考慮交叉編譯複雜度)

## 決策流程圖

```
開始
  ↓
是否為計算密集型? ──否→ 保留 Go
  ↓ 是
資料大小 > 100KB? ──否→ 執行 benchmark
  ↓ 是
CPU 使用率 > 15%? ──否→ 中優先級
  ↓ 是
是否有 SIMD 機會? ──是→ 高優先級 (轉換 Rust)
  ↓ 否
GC 壓力高? ──是→ 高優先級 (轉換 Rust)
  ↓ 否
中優先級 (benchmark 後決定)
```

## 實際案例參考

### 案例 1: 路徑規劃演算法

- **場景**: 機器人導航，A* 演算法
- **資料大小**: 地圖 1000x1000 節點
- **CPU 使用率**: 35%
- **決策**: ✅ 轉換為 Rust
- **結果**: 4.1x 加速

### 案例 2: JSON 解析

- **場景**: API 回應解析
- **資料大小**: 平均 50KB
- **CPU 使用率**: 3%
- **決策**: ❌ 保留 Go
- **原因**: CGO 開銷 > 效能增益

### 案例 3: 影像濾鏡

- **場景**: 即時影像處理
- **資料大小**: 1920x1080 像素
- **CPU 使用率**: 28%
- **決策**: ✅ 轉換為 Rust (SIMD)
- **結果**: 6.8x 加速
