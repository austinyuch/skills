# 模板變量映射表

## 说明

本文件列出了從通用六階段工作流模板（PROMPT_TEMPLATE.md）產生具體Workflow Skill时需要替换的所有變量及其替换規則。

## 變量分类

这些變量定義了Skill的基本身份和定位。

| 變量名 | 描述 | 示例值 | 来源 |
|--------|------|--------|------|

| `{角色名}` | Skill的角色名称 | Writer、PRD產生器、Kiro | requirements.md - 角色名称 |
| `{專業领域}` | 具體專業领域 | 写作、產品管理、開發 | requirements.md - 專業领域 |

## 變量替换流程

### 步骤1：提取基础信息

從 `requirements.md` 提取：
- 领域、角色、使用者相关變量
- 輸入輸出類型
- 價值主张

### 步骤2：提取流程設計

從 `workflow_design.md` 提取：
- 六階段的所有變量
- 評估維度和权重
- 确认方式

### 步骤3：系統化替换

1. 先替换基础變量（领域名、角色名等）
2. 再替换六階段變量
3. 然后替换評估變量
4. 最后替换辅助變量

### 步骤4：语义檢查

- 確保替换后语句通顺
- 檢查是否有遗漏的變量
- 驗證邏輯连贯性

## 使用示例

### 示例1：替换开场白

**模板原文**：
```
您好！我是{角色名}，您的AI{專業领域}助手。我将幫助您将{輸入類型}系統化地转變为高品質的{輸出類型}。
```

**Writer案例替换**：
```
您好！我是Writer，您的AI写作助手。我将幫助您将写作想法系統化地转變为高品質的文章內容。
```

**Kiro案例替换**：
```
您好！我是Kiro，您的AI開發助手。我将幫助您将功能想法系統化地转變为高品質的Laravel后端代码實作。
```

### 示例2：替换階段描述

**模板原文**：
```
1. **{階段1名称}** - {階段1描述}
2. **{階段2名称}** - {階段2描述}
```

**Writer案例替换**：
```
1. **內容构思** - 明确写作主题和核心思路
2. **大纲設計** - 規劃文章結構和邏輯框架
```

**PRD案例替换**：
```
1. **需求理解** - 深度理解產品需求和業務背景
2. **方案設計** - 設計產品功能和使用者體驗
```

### 示例3：替换評估維度

**模板原文**：
```
- **{維度1}評估** ({权重1}%权重)：{評估內容1}
- **{維度2}評估** ({权重2}%权重)：{評估內容2}
```

**Writer案例替换**：
```
- **內容品質評估** (30%权重)：准确性、深度、價值
- **結構邏輯評估** (25%权重)：組織、流畅、连贯
```

## 常见問題

### Q1: 如何確定目錄名？

**A**: 根据Skill的用途：
- 內容创作类：writers、content、articles
- 產品管理类：prd、requirements、products
- 技術開發类：specs、code、development
- 資料分析类：analysis、data、reports

### Q2: 評估維度應该如何設計？

**A**: 根据领域特征選擇3-5個維度：
- 內容类：品質、結構、語言、體驗
- 技術类：需求符合度、設計一致性、代码規範、代码品質
- 產品类：准确性、完整性、可行性、创新性

### Q3: 确认问句如何設計？

**A**: 應该：
- 自然、友好，避免机械
- 總結當前階段成果
- 引導进入下一階段
- 给使用者選擇权

**好的示例**：
- "构思清晰了，准备好設計大纲了吗？"
- "需求分析完成，我们可以进入設計階段了吗？"

**不好的示例**：
- "是否繼續？"（太简單）
- "请确认是否进入下一階段。"（太机械）

## 總結

變量映射是從通用模板產生具體Skill的核心環節。透過系統化的變量替换，可以快速建立專業、規範的Workflow Skill，同时保证品質和一致性。
