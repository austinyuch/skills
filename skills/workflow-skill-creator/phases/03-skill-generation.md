# 第三階段：Skill完整產生

## 階段概述

基於前两階段的需求分析和流程設計，產生完整的Workflow Skill，包括SKILL.md主文件、phases階段指南、reference參考文件等所有必要文件。

## 觸發條件

- 第二階段的 `workflow_design.md` 已完成并获得使用者认可
- 或使用者明确表示要產生Skill

## 階段目标

1. 基於通用模板进行系統化的變量替换
2. 產生符合Agent Skill規範的SKILL.md主文件
3. 產生六個階段的詳細實施指南文件（phases目錄）
4. 產生必要的參考文件（reference目錄）
5. 確保所有文件符合Agent Skill標準

## 具體工作內容

#### YAML前置內容要求

- **name**: 必須是kebab-case格式，只包含小写字母、数字和连字符，最多64字符
- **description**: 清晰描述功能和使用时机，最多1024字符，應包含：
  - 领域描述
  - 主要功能
  - 使用場景（"當使用者需要...时使用"）
  - 适用場景（"適用於..."）

## 重要約束

1. **Agent Skill規範遵循**
   - SKILL.md必須包含正确的YAML前置內容
   - name必須是kebab-case格式
   - description必須清晰描述功能和使用时机
   - 文件結構必須符合標準

2. **模板變量完全替换**
   - 所有{變量}必須被替换为實際值
   - 不能遗留任何未替换的變量
   - 替换后的內容必須语义通顺

3. **內容一致性**
   - SKILL.md、phases文件、reference文件之间必須一致
   - 階段名称、文件名、描述必須统一
   - 評估維度在所有相关文件中必須一致

4. **專業化定制**
   - 必須针對具體领域进行專業化表达
   - 使用该领域的專業术语和行業規範
   - 示例必須贴近實際使用場景

5. **使用者确认机制**
   - 產生完成后必須请使用者预覽
   - 如果使用者要求修改，必須調整后再次确认
   - 必須繼續反馈-修订循环直到获得明确批准

## 產生流程

### 步骤1：建立目錄結構

```bash
mkdir -p .opencode/skills/{skill-name}/{phases,reference}
```

### 步骤2：產生SKILL.md

1. 基於通用模板
2. 替换所有變量
3. 專業化定制內容
4. 添加正确的YAML前置內容

### 步骤3：產生六個Phase文件

1. 基於workflow_design.md中的階段設計
2. 为每個階段建立詳細指南
3. 包含完整的示例和模板
4. 第4階段必須包含完整評估体系

### 步骤4：產生Reference文件

1. 复制PROMPT_TEMPLATE.md
2. 建立VARIABLE_MAPPING.md
3. 根据需要添加领域參考文件

### 步骤5：產生README.md

1. 编写清晰的使用說明
2. 列出文件結構
3. 提供示例

### 步骤6：品質檢查

1. 檢查YAML格式正确性
2. 檢查所有變量已替换
3. 檢查文件间一致性
4. 檢查markdown格式

## 完成标志

當满足以下條件时，第三階段完成：

1. ✅ 已建立所有必需文件（SKILL.md、README.md、6個phase文件）
2. ✅ YAML前置內容格式正确
3. ✅ 所有模板變量已替换
4. ✅ 文件间內容一致
5. ✅ 使用者明确认可產生的Skill
