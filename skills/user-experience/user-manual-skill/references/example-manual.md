## 2. UX Flow: 採購單處理流程

### 建立採購單
**情境描述**: 採購專員每週需彙整需求，並在系統中建立對應的採購單。

1. **進入採購單列表**
   - 點擊左側導覽列的「採購管理」>「採購單列表」。
   - *(預期截圖: `docs/manual/assets/order-flow-01-list.png` - 顯示包含「新增採購單」按鈕的空列表或現有列表)*
   
2. **填寫採購單內容**
   - 點擊右上角的「新增採購單」按鈕，進入填寫頁面。
   - *(預期截圖: `docs/manual/assets/order-flow-02-create.png` - 顯示空白的採購單表單)*
   - 依序填寫「供應商名稱」、「預計交期」與「品項明細」。
   
3. **送出審核**
   - 確認資料無誤後，點擊右下角的「送出審核」按鈕。
   - *(預期截圖: `docs/manual/assets/order-flow-03-submit-success.png` - 顯示送出成功的 Toast 提示與狀態變更為「待審核」的列表頁)*

## 3. 功能頁面詳解

### 採購單列表頁 (List Page)

*(預期截圖: `docs/manual/assets/feature-list-view.png` - 包含篩選器、資料表與分頁元件)*

**主要功能說明**:
- **篩選器 (A)**: 可依據「狀態」(草稿、待審核、已核准) 與「建立日期」進行篩選。
- **匯出按鈕 (B)**: 點擊後可將當前篩選結果匯出為 Excel 檔案。
- **單行操作 (C)**: 每筆資料右側提供「檢視詳情」與「刪除」(僅限草稿狀態) 的快捷操作。

### 採購單詳情頁 (Detail Page)

*(預期截圖: `docs/manual/assets/feature-detail-view.png` - 顯示單筆採購單的所有唯讀資訊與審核按鈕)*

**主要功能說明**:
- **基本資訊區塊 (A)**: 顯示訂單編號、供應商與總金額。
- **狀態列 (B)**: 顯示目前的審核進度（例如：主管簽核中）。
- **審核操作 (C)**: (僅限主管權限可見)
  - `核准`: 點擊後訂單狀態變更為「已核准」並通知供應商。
  - `退回`: 點擊後需填寫退回原因，訂單狀態變更為「已退回」。

> ⚠️ **注意**：一旦訂單被核准，將無法再進行編輯或刪除操作。
