# B端供應鏈PRD產生器 Agent Skill

專業的B端供應鏈產品需求文件產生助手，深度理解採購、庫存、物流、供應商管理等核心業務場景。

## 功能概述

本Skill幫助您系統化地建立高品質的B端供應鏈PRD文件，透過六個連續的階段確保從需求調研到最終交付的全過程品質。

### 核心特性

- **B端供應鏈专精**：深度理解供應鏈複雜性和行業特點
- **六階段工作流**：需求調研→產品規劃→文件撰寫→需求評審→文件優化→發布准备
- **品質保障**：多維度評審机制，確保PRD品質达到8.5分以上
- **敏捷適配**：支援功能增量迭代和快速交付
- **現有系統增強**：預設基於現有系統功能增強（非全新開發）

## 六階段工作流

### 1. 需求調研

深度分析供應鏈業務背景、現有系統现状、使用者角色需求。

**輸出**：`.opencode/prd/{功能名}/research.md`

**關鍵內容**：

- 業務背景分析
- 現有系統評估
- 使用者角色研究
- 功能增量定義
- 竞品分析

### 2. 產品規劃

基於需求調研结果制定產品架構、功能模組和優先順序規劃。

**輸出**：`.opencode/prd/{功能名}/architecture.md`

**關鍵內容**：

- 產品整體架構
- 功能清單与優先順序
- 使用者體驗設計
- 版本迭代计划

### 3. 文件撰寫

按照標準PRD模板建立完整的需求文件。

**輸出**：`.opencode/prd/{功能名}/draft.md`

**關鍵內容**：

- 版本迭代規劃
- 業務背景与目标
- 業務名词定義
- 流程圖
- 功能需求详述与介面設計
- 資料模型
- 驗收標準

### 4. 需求評審

多維度評估PRD品質，重点关注可執行性。

**輸出**：`.opencode/prd/{功能名}/review.md`

**評估維度**：

- 需求完整性（25%）
- 可執行性（25%）
- 使用者價值（20%）
- 業務邏輯（20%）
- 文件品質（10%）

### 5. 文件優化

基於評審報告優化PRD，提升品質到8.5分以上。

**輸出**：更新draft.md或建立prd.md，以及optimization_log.md

**優化控制**：

- 最多3轮優化避免无限循环
- 每轮優化后重新評分
- 达到8.5分或使用者满意即停止

### 6. 發布准备

優化格式，制作最終可交付的PRD文件。

**輸出**：`.opencode/prd/{功能名}/final.md` + README.md + images/

**優化內容**：

- 文件結構和格式規範
- 圖表顯示效果
- 版本信息完善
- 建立交付目錄

## 适用場景

本Skill特别適用於：

- **B端供應鏈系統**：採購、庫存、物流、供應商管理
- **現有系統增強**：基於現有系統的功能迭代和優化

## 專業要求

### 供應鏈專業术语

Skill会使用供應鏈行業標準术语：

- 採購管理：採購申请、採購订單、供應商評估、合同管理
- 庫存管理：庫存周转、安全庫存、ABC分类、庫存预警
- 物流管理：運輸管理、配送路線、運費結算、貨物跟踪
- 供應商协同：订單协同、发货通知、對账結算、績效評估

详见 [供應鏈業務术语表](./reference/SUPPLY_CHAIN_TERMS.md)

### 角色定位

本Skill作为專業產品經理角色：

- 专注業務價值和使用者體驗
- 不涉及技術實作細節
- 不涉及資料庫表SQL設計
- 描述清楚業務邏輯和資料實體即可

### 敏捷開發適配

PRD文件支援敏捷開發：

- **功能原子化**：每個功能点可独立開發、測試、上線
- **迭代價值**：每個2-4周迭代都能交付可用業務價值
- **集成友好**：新功能与現有系統集成影響最小化
- **監控完备**：關鍵業務指标的实时監控和告警

## 品質保障

### 評審机制

采用五維度評審体系：

| 維度       | 权重 | 说明                               |
| ---------- | ---- | ---------------------------------- |
| 需求完整性 | 25%  | 功能覆盖度、边界條件、异常處理     |
| 可執行性   | 25%  | 開發理解度、資源匹配度、驗收標準   |
| 使用者價值 | 20%  | 需求匹配度、體驗設計、價值创造     |
| 業務邏輯   | 20%  | 流程合理性、規則完整性、資料一致性 |
| 文件品質   | 10%  | 結構清晰度、描述准确性、格式規範性 |

详见 [PRD評審標準](./reference/REVIEW_CRITERIA.md)

### 品質分級

- **≥8.5分**：优秀，可直接进入發布准备
- **7.0-8.4分**：良好，建議優化后發布
- **6.0-6.9分**：需要優化，建議優化主要問題
- **<6.0分**：必須優化，需要大幅改进

## 常见問題

### Q: Skill什么时候会被觸發？

A: 當您提到以下關鍵词时，Skill会自動觸發：

- "PRD"、"產品需求文件"
- "供應鏈"、"採購"、"庫存"、"物流"、"供應商"
- "功能需求"、"產品規劃"

### Q: 如何跳到特定階段？

A: Skill会自動识别當前階段，如果想跳到特定階段，可以明确说明：

- "我要修改需求調研"
- "直接开始文件撰寫"
- "重新进行評審"

### Q: 如何繼續未完成的PRD？

A: 说"繼續上次任務"，Skill会自動檢查并繼續。

### Q: PRD應该写到什么程度的細節？

A: PRD應该詳細到：

- 開發人员能理解要做什么功能
- 設計师能設計介面原型
- 測試人员能编写測試用例
- 產品运营能理解業務邏輯

但不應该包括技術實作細節和資料庫設計。

### Q: 如何處理需求變更？

A: 可以随时返回任何階段进行修改，Skill会保留現有工作成果并更新相关文件。
