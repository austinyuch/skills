# 第六階段：發布准备

## 階段目标

優化格式，制作最終可交付的PRD文件，確保文件达到發布標準并便于团队使用。

## 前置條件

- PRD文件品質已达到8.5分以上，或使用者明确满意當前版本
- 存在 `.opencode/prd/{功能名}/draft.md` 或 `prd.md` 文件
- 所有主要問題已解决

## 工作內容

### 1. 優化文件結構和格式規範

#### 统一格式規範

- **標題层級**：確保標題层級正确，H1→H2→H3邏輯清晰
- **列表格式**：统一使用有序列表或无序列表
- **表格格式**：確保表格對齐，內容完整
- **代码块**：统一代码块的語言标识

#### 完善目錄結構

```markdown
# 供應鏈系統PRD：{功能名} V1.0

## 目錄

1. [版本迭代規劃](#1-版本迭代規劃)
2. [業務背景与目标](#2-業務背景与目标)
   2.1 [項目背景](#21-項目背景)
   2.2 [核心使用者与場景](#22-核心使用者与場景)
   2.3 [業務目标](#23-業務目标)
3. [業務名词](#3-業務名词)
4. [流程圖](#4-流程圖)
5. [功能需求详述与介面設計](#5-功能需求详述与介面設計)
6. [資料模型](#6-資料模型)
7. [驗收標準](#7-驗收標準)
```

#### 優化视觉呈现

- **使用分隔線**：在重要章節之间使用分隔線
- **突出重点信息**：使用粗体、引用块突出重点
- **统一圖标**：使用统一的圖标标识（📋、⚠️、✅等）
- **代码高亮**：確保代码块有正确的语法高亮

### 2. 完善圖表顯示效果

#### 檢查Mermaid圖表

確保所有Mermaid圖表：

- 语法正确，可以正常渲染
- 布局清晰，易于理解
- 标签准确，无拼写错误
- 颜色搭配合理（如需要）

#### 檢查圖片占位符

確保所有圖片占位符：

- 路径正确，格式统一
- 有清晰的说明文本
- 有插入提示信息

```markdown
![供應商管理頁面](./images/supplier-management.png)

> 📋 **提示**：粘贴供應商管理頁面原型圖，包含筛选区、列表区和操作区
```

#### 優化表格展示

- 確保表格內容對齐
- 适當使用表格合并
- 表格不要过宽，必要时拆分
- 表格標題清晰明确

### 3. 完善版本信息

#### 添加文件元信息

```markdown
---
文件標題: 供應鏈系統PRD：{功能名}
文件版本: V1.0
建立日期: YYYY-MM-DD
最后更新: YYYY-MM-DD
文件狀態: 已發布
負責人: [產品經理姓名]
評審人: [評審人姓名]
---
```

#### 添加變更歷史

```markdown
## 變更歷史

| 版本 | 日期       | 變更內容   | 變更人 |
| ---- | ---------- | ---------- | ------ |
| V1.0 | YYYY-MM-DD | 初始版本   | [姓名] |
| V1.1 | YYYY-MM-DD | [變更描述] | [姓名] |
```

### 4. 建立最終PRD文件

建立 `.opencode/prd/{功能名}/final.md` 最終版本：

- 复制優化后的PRD內容
- 應用所有格式優化
- 完善版本信息
- 添加文件元信息
- 檢查所有連結和引用

### 5. 建立交付目錄和文件

#### 目錄結構

```
.opencode/prd/{功能名}/
├── final.md              # 最終PRD文件
├── research.md          # 需求調研報告（附件）
├── architecture.md      # 產品規劃文件（附件）
├── review.md            # 評審報告（附件）
├── optimization_log.md  # 優化記錄（附件）
├── README.md            # 文件阅读指南
└── images/              # 圖片資源目錄
    ├── .gitkeep        # 保持目錄
    └── README.md       # 圖片说明
```

#### 建立阅读指南

建立 `.opencode/prd/{功能名}/README.md`：

```markdown
# {功能名}產品需求文件

## 文件概覽

本目錄包含{功能名}的完整產品需求文件及相关附件。

## 文件列表

### 核心文件

- **final.md** - 最終PRD文件（主文件）
  - 版本：V1.0
  - 狀態：已發布
  - 用途：開發、設計、測試的主要參考文件

### 附件文件

- **research.md** - 需求調研報告
  - 用途：瞭解業務背景和使用者需求

- **architecture.md** - 產品規劃文件
  - 用途：瞭解產品架構和功能規劃

- **review.md** - 需求評審報告
  - 用途：瞭解PRD品質評估和主要改进点

- **optimization_log.md** - 優化記錄
  - 用途：瞭解文件優化歷程

## 快速开始

### 產品經理

1. 阅读 `final.md` 瞭解完整需求
2. 參考 `architecture.md` 瞭解產品架構
3. 查看 `review.md` 瞭解品質評估

### 開發人员

1. 重点阅读 `final.md` 第5章"功能需求详述"
2. 參考 `final.md` 第6章"資料模型"
3. 參考 `final.md` 第7章"驗收標準"

### 設計师

1. 重点阅读 `final.md` 第5章"介面設計"
2. 參考 `final.md` 第4章"流程圖"
3. 參考 `architecture.md` 瞭解整體架構

### 測試人员

1. 重点阅读 `final.md` 第7章"驗收標準"
2. 參考 `final.md` 第5章"業務規則"
3. 參考 `final.md` 第4章"流程圖"

```

#### 建立圖片目錄说明

建立 `.opencode/prd/{功能名}/images/README.md`：

```markdown
# 圖片資源目錄

本目錄用于存放PRD文件中引用的所有圖片資源。

## 圖片命名規範

- 使用kebab-case命名：`supplier-management-page.png`
- 包含模組前缀：`supplier-` `inquiry-` `price-`
- 说明圖片類型：`-list` `-form` `-flow` `-model`

## 圖片類型

### 頁面原型圖

- `supplier-management-list.png` - 供應商列表頁
- `supplier-management-form.png` - 供應商表單頁
- `inquiry-collaboration-list.png` - 詢价列表頁

### 流程圖

- `supplier-admission-flow.png` - 供應商准入流程
- `inquiry-collaboration-flow.png` - 詢价协同流程

### 資料模型圖

- `supplier-data-model.png` - 供應商資料模型
- `inquiry-data-model.png` - 詢价資料模型

## 圖片格式要求

- **格式**：PNG或JPG
- **分辨率**：至少1920x1080
- **大小**：單個文件不超过5MB
- **清晰度**：確保文字清晰可读

## 添加新圖片

1. 按照命名規範命名文件
2. 将文件放入本目錄
3. 在PRD文件中引用
4. 更新本README文件
```
