# 第一階段：需求調研

## 階段目标

深度分析供應鏈業務背景、現有系統现状、使用者角色需求，为產品規劃奠定坚实的基础。

## 工作內容

### 1. 調研供應鏈行業標準和最佳實務

- 研究供應鏈行業发展趋势和竞争環境
- 分析同类產品的功能特點和使用者體驗
- 瞭解行業標準流程和最佳實務
- 识别供應鏈管理的核心痛点和效率瓶颈

### 2. 分析現有系統功能評估和集成现状

- 梳理現有系統的功能模組和架構
- 評估現有系統的資料結構和接口
- 分析現有系統的業務流程和使用者角色
- 识别現有系統的优势和不足

### 3. 建立需求調研文件

建立 `.opencode/prd/{功能名}/research.md`，包含以下內容：

#### 3.1 業務背景分析

- **供應鏈痛点**：當前業務面临的主要問題和挑戰
- **效率瓶颈**：影響業務效率的關鍵因素
- **成本控制难点**：成本管理中的突出問題
- **業務发展瓶颈**：阻碍業務发展的核心障碍

#### 3.2 現有系統評估

- **功能模組**：現有系統的核心功能和模組划分
- **資料結構**：關鍵業務實體和資料关系
- **集成接口**：与其他系統的集成方式和接口
- **技術架構**：系統的技術栈和架構設計（仅作參考，PRD不涉及技術細節）

#### 3.3 供應鏈角色研究

针對不同角色建立使用者画像：

| 角色     | 职责                 | 使用場景             | 關鍵痛点                   |
| -------- | -------------------- | -------------------- | -------------------------- |
| 採購专员 | 供應商管理、订單執行 | 供應商評估、採購下單 | 供應商信息分散、詢价效率低 |
| 仓储主管 | 庫存管理、收发货     | 入庫驗收、庫存盘点   | 庫存資料不准确、盘点繁琐   |
| 物流专员 | 運輸协调、物流跟踪   | 運輸安排、貨物跟踪   | 物流信息滞后、异常處理难   |
| 财务专员 | 對账結算、成本核算   | 採購對账、费用結算   | 對账效率低、資料不准确     |
| 供應商   | 订單接收、发货管理   | 订單确认、发货通知   | 信息不透明、沟通成本高     |

#### 3.4 功能增量定義

- **現有功能評估**：梳理現有功能的完整性和有效性
- **增強范围**：明确本次功能增強的范围和边界
- **边界说明**：清晰定義哪些在范围内，哪些不在范围内
- **集成方式**：新功能如何与現有系統集成

### 4. 竞品分析

- 分析主要竞品的功能特點
- 對比竞品的优势和劣势
- 提取可借鉴的設計思路
- 识别差异化竞争点

## 完成确认

當調研文件建立完成后，我会：

1. 向您展示需求調研報告的核心內容
2. 征求您對調研分析的意见和建議
3. 确认調研分析是否准确反映了實際情况
4. 等待您的明确认可后进入下一階段

## 注意事項

- 需求調研階段要关注業務價值，而非技術實作
- 要深入理解使用者痛点，而非仅仅列出功能清單
- 要基於現有系統进行增強設計，而非從零开始
- 要考虑敏捷迭代，而非一次性交付所有功能
- 調研要客观全面，为后续階段提供充分依据

## 示例片段

### 業務背景分析示例

**供應鏈痛点**：

- 供應商信息分散在各個Excel表格中，查詢困难，信息更新不及时
- 採購詢价主要透過邮件和电话，效率低下，价格對比複雜
- 缺乏统一的供應商評估標準，品質管控存在风險
- 業務量增长30%，現有人工流程已无法满足效率要求

**效率瓶颈**：

- 詢价周期平均5天，严重影響採購回應速度
- 供應商信息查詢耗时30分钟，效率低下
- 价格對比需要人工整理，容易出错
- 供應商績效評估缺乏量化工具，主观性強

### 角色研究示例

| 角色     | 职责                 | 使用場景                 | 關鍵痛点             |
| -------- | -------------------- | ------------------------ | -------------------- |
| 採購专员 | 供應商開發、詢价下單 | 供應商信息查詢、价格比较 | 信息分散，詢价耗时长 |
| 採購主管 | 供應商評估、策略制定 | 供應商績效分析、准入審批 | 缺乏量化評估工具     |
| 质检员   | 供應商品質监督       | 品質問題反馈、整改跟踪   | 品質資料难以统计分析 |

### 功能增量定義示例

**現有功能評估**：

- 現有系統支援基础的供應商信息管理
- 已實作简單的詢价流程
- 缺少价格對比和供應商評估功能

**增強范围**：

- 完善供應商信息管理功能，支援多維度筛选和查詢
- 優化詢价流程，實作在線协同詢价
- 新增价格對比分析功能
- 新增供應商評級和績效管理功能

**边界说明**：

- 范围内：供應商管理、詢价协同、价格對比、績效評估
- 范围外：合同管理、财务結算、物流跟踪
- 集成方式：透過API与ERP、财务系統集成
