# 第五階段：文件優化

## 階段目标

基於評審報告優化PRD，提升品質到8.5分以上，確保文件达到發布標準。

## 前置條件

- 已完成需求評審階段
- 存在 `.opencode/prd/{功能名}/review.md` 文件
- 評審報告顯示得分在6.0-8.4分之间，或使用者明确要求優化

## 優化原則

### 1. 優先順序驱動

- **P0問題优先**：必須修复的严重問題
- **P1問題其次**：重要但不阻塞的問題
- **P2問題最后**：锦上添花的改进

### 2. 影響范围評估

- **高影響**：影響核心功能或多個模組
- **中影響**：影響單個功能或模組
- **低影響**：影響局部細節

### 3. 工作量控制

- **每轮優化目标**：解决主要問題，提升1-1.5分
- **最多優化轮次**：3轮，避免无限循环
- **时间控制**：每轮優化控制在合理时间内

## 工作內容

### 1. 制定優化计划

基於評審報告，按優先順序制定優化计划：

**第一轮優化重点**（目标：解决P0問題）：

- 列出所有P0級别問題
- 評估修复工作量
- 制定修复方案
- 確定優化顺序

**第二轮優化重点**（目标：解决P1問題）：

- 列出所有P1級别問題
- 評估修复工作量
- 制定修复方案
- 選擇性修复

**第三轮優化重点**（目标：提升文件品質）：

- 處理部分P2問題
- 完善細節描述
- 優化文件格式
- 提升整體品質

### 2. 執行優化改进

针對評審報告中的問題，逐項进行優化：

#### 需求完整性優化

- **补充缺失功能**：添加遗漏的功能場景
- **完善边界條件**：明确功能边界和約束
- **增強异常處理**：补充异常場景的處理規則

#### 可執行性優化

- **清晰化需求描述**：消除歧义，增強准确性
- **完善介面設計**：补充介面設計細節
- **明确業務規則**：詳細定義業務規則和驗證邏輯

#### 使用者價值優化

- **突出使用者價值**：明确每個功能的價值和ROI
- **優化使用者體驗**：简化流程，提升易用性
- **強化價值创造**：量化業務收益

#### 業務邏輯優化

- **優化業務流程**：简化流程，提升效率
- **完善業務規則**：补充規則定義，消除冲突
- **增強資料一致性**：優化資料模型，保证一致性

#### 文件品質優化

- **優化文件結構**：調整章節組織，提升可读性
- **提升描述准确性**：修正术语使用，消除错误
- **規範文件格式**：统一格式，完善圖表

### 3. 更新PRD文件

将優化內容更新到PRD文件：

- 修改 `.opencode/prd/{功能名}/draft.md` 文件
- 或建立 `.opencode/prd/{功能名}/prd.md` 新版本
- 記錄優化變更內容
- 保持与前序文件的一致性

### 4. 重新評審

每轮優化完成后，重新評審PRD品質：

- 按照評審標準重新評分
- 對比優化前后的得分變化
- 识别剩余問題
- 决定是否需要繼續優化

### 5. 建立或更新優化記錄

建立或更新 `.opencode/prd/{功能名}/optimization_log.md`，記錄優化歷程：

```markdown
# {功能名}優化記錄

## 優化歷史

### 第一轮優化（YYYY-MM-DD）

**優化前得分**：7.2分

**主要問題**：

1. 供應商評級算法未明确
2. 詢价超时處理規則不清晰
3. 价格對比維度不够全面

**優化內容**：

1. 补充供應商評級算法的計算規則
   - 定義了品質、交期、价格、服务四個維度
   - 明确了各維度的权重分配
   - 说明了評級的計算公式

2. 完善詢价超时處理規則
   - 定義超时时间为48小时
   - 超时后自動关闭詢价單
   - 发送提醒通知给相关人员

3. 扩展价格對比維度
   - 增加品質評級對比
   - 增加交期對比
   - 增加歷史合作記錄對比

**優化后得分**：8.1分

**得分提升**：+0.9分

**剩余問題**：

1. 统计报表需求描述过于简單
2. 部分業務規則定義不够詳細

---

### 第二轮優化（YYYY-MM-DD）

[記錄第二轮優化內容]

---

## 優化總結

- **初始得分**：7.2分
- **當前得分**：8.5分
- **总提升**：+1.3分
- **優化轮次**：2轮
- **达标情况**：已达到發布標準（≥8.5分）
```

## 優化控制

### 最多優化轮次

- **第一轮**：解决P0問題，目标得分7.5-8.0
- **第二轮**：解决P1問題，目标得分8.0-8.5
- **第三轮**：完善細節，目标得分≥8.5

超过3轮仍未达标，需要：

- 重新審视需求調研和產品規劃
- 考虑是否需要調整產品方案
- 与使用者讨论是否降低標準或分階段實施

### 停止優化條件

满足以下任一條件即可停止優化：

1. **得分达标**：综合得分≥8.5分
2. **使用者满意**：使用者明确表示满意并认可當前版本
3. **轮次上限**：已完成3轮優化
4. **边际效益递减**：優化效果不明显，提升<0.3分

## 品質檢查清單

每轮優化完成后，檢查：

- [ ] P0問題是否全部修复
- [ ] P1問題是否大部分修复
- [ ] 優化內容是否准确完整
- [ ] 是否保持了文件一致性
- [ ] 得分是否有明显提升
- [ ] 是否达到了優化目标
- [ ] 是否需要繼續優化

## 完成确认

每轮優化完成后，我会：

1. 向您展示優化內容和效果
2. 说明得分變化和提升幅度
3. 列出剩余問題和改进空间
4. 征求您的意见和建議
5. 确认是否需要繼續優化
6. 等待您的明确决定

## 注意事項

- 優化要基於評審報告的具體問題
- 要保持与前序文件的一致性
- 不要过度優化导致范围蔓延
- 控制優化轮次避免无限循环
- 每轮優化都要重新評審驗證效果

## 優化示例

### 優化前（問題）

**供應商評級功能描述**：

```markdown
系統支援供應商評級功能，採購人员可以對供應商进行評級。
```

**問題**：

- 評級規則未明确
- 評級維度不清楚
- 評級结果如何使用未说明

### 優化后（改进）

**供應商評級功能描述**：

```markdown
### 供應商評級功能

**功能概述**：
系統支援多維度的供應商評級功能，幫助企业量化評估供應商績效，为採購决策提供資料支援。

**評級維度**：

| 維度 | 权重 | 評分標準                     | 資料来源 |
| ---- | ---- | ---------------------------- | -------- |
| 品質 | 35%  | 合格率、退货率、品質問題次数 | 质检系統 |
| 交期 | 30%  | 准时交货率、延迟天数         | 订單系統 |
| 价格 | 25%  | 价格竞争力、成本控制         | 採購系統 |
| 服务 | 10%  | 回應速度、配合度、問題處理   | 人工評分 |

**評級計算**：

综合得分 = 品質得分 × 35% + 交期得分 × 30% + 价格得分 × 25% + 服务得分 × 10%

**評級等級**：

- A級（优秀）：综合得分 ≥ 90分
- B級（良好）：综合得分 80-89分
- C級（合格）：综合得分 70-79分
- D級（观察）：综合得分 < 70分

**評級應用**：

- A級供應商：优先合作，增加订單份额
- B級供應商：正常合作，稳定订單份额
- C級供應商：限制合作，要求改进计划
- D級供應商：停止新订單，启動替换程序

**評級周期**：

- 評級周期：每季度評估一次
- 資料统计期：最近6個月的業務資料
- 評級有效期：3個月，到期自動重新評估
```

**改进点**：

- 明确了評級的4個維度和权重
- 詳細说明了評級計算規則
- 定義了評級等級和標準
- 说明了評級结果的應用方式
- 补充了評級周期和資料来源
