# 階段四：程式碼產生階段詳細指南

## 階段目標

產生完整的、可執行的靜態互動HTML程式碼。

## 輸入條件

- 介面規劃檔案（`ui_plan.md`）已完成
- 模板選擇記錄（`template_choice.md`）已完成
- 模板檔案已載入

## 頁面產生原則

1. 如果需要產生一組多個頁面
    - 必須有一個index.html檔案，作為主頁面串联所有頁面。
    - 每個頁面必須有獨立的HTML檔案。
    - 所有頁面必須使用相同的UI模板風格。
    - 注意多個頁面之间的跳轉。
    - 多個頁面必須邏輯闭环，支援点击頁面内的按钮或連結跳轉到其他頁面。

## 程式碼產生原則

### 核心原則

1. 嚴格參考已选模板的樣式和組件結構，除非使用者有特殊要求，不要進行修改。
2. 必須完整讀取模板檔案，並嚴格按照模板檔案的結構和樣式產生程式碼。如果模板檔案太长，需要分多次讀取。
3. 圖標优先使用FontAwesome开源圖標庫，非必要不使用圖標，避免滥用。
4. 嚴格遵循介面規劃，按照ui_plan.md中的設計實作。
5. 使用模板樣式系統，基於选定模板的樣式和組件。
6. 程式碼結構清晰，HTML/CSS/JS分离清晰，層次分明。
7. 詳細程式碼註解，關鍵邏輯必須有註解說明。
8. 語义化HTML，使用語义化標籤，提升可读性。
9. 回應式設計，支援不同屏幕尺寸。
10. 直接可執行，無需建置工具，浏覽器直接打开即可執行。

## 產生步骤

### 步骤1：准备產生環境

建立輸出目錄：

```bash
.opencode/ui-generator/{功能名}/generated/
```

載入模板資源：

- CSS樣式系統
- JavaScript互動框架
- 組件程式碼片段

### 步骤2：逐頁面產生HTML

按照介面規劃中的頁面清單，逐個產生HTML檔案。

#### HTML檔案結構

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>[頁面標題]</title>
    <style>
        /* [模板的CSS樣式] */
    </style>
</head>
<body>
    <!-- [頁面內容] -->
    
    <script>
        // [JavaScript互動邏輯]
    </script>
</body>
</html>
```

**資料展示**：

**篩選搜尋**：

- ✅ 搜尋功能
- ✅ 篩選條件
- ✅ 清除篩選
- ✅ 分頁

**CRUD操作**：

- ✅ 新增（模態框 + 表格單 + 儲存）
- ✅ 編輯（模態框 + 表格單填充 + 更新）
- ✅ 檢視詳情（模態框 + 資料展示）
- ✅ 刪除（確認對话框 + 刪除）
- ✅ 批量操作（如需要）

**表格單驗證**：

- ✅ 必填項驗證
- ✅ 格式驗證
- ✅ 錯誤提示顯示

**狀態管理**：

- ✅ 狀態標籤顯示
- ✅ 狀態切换（如需要）

**互動反馈**：

- ✅ 成功/失败訊息提示
- ✅ 載入動画
- ✅ 確認對话框

**其他功能**：

- ✅ 匯入匯出（如需要）
- ✅ 其他特殊功能

**HTML註解**：

```html

<div class="stats-cards">
    <!-- 卡片內容 -->
</div>

<!-- 篩選搜尋区 -->
<div class="table-filter">
    <!-- 篩選表格單 -->
</div>
```

**JavaScript註解**：

```javascript
// ========== 資料定義 ==========
const mockData = [...];

// ========== 基础功能函數 ==========
// 搜尋清單格

    // 實作邏輯
}

// ========== CRUD操作函數 ==========
// 新增資料

    // 實作邏輯
}
```

## 評估維度

- **程式碼完整性（30%）**：是否實作了所有規劃的功能
- **程式碼規範性（25%）**：是否符合HTML/CSS/JS規範
- **程式碼可读性（20%）**：結構是否清晰，註解是否完善
- **視觉还原度（15%）**：是否符合模板風格
- **功能可用性（10%）**：互動是否正常工作
