# 階段二：介面規劃階段詳細指南

## 階段目標

規劃詳細的頁面佈局、功能模組分布和互動流程。

## 輸入條件

- 需求理解檔案（`input.md`）已完成
- 使用者已確認需求理解准确

## 頁面產生原則

1. 如果需要產生一組多個頁面
   - 必須有一個index.html檔案，作為主頁面串联所有頁面。
   - 每個頁面必須有獨立的HTML檔案。
   - 所有頁面必須使用相同的UI模板風格。
   - 注意多個頁面之间的跳轉。
   - 多個頁面必須邏輯闭环，支援点击頁面内的按钮或連結跳轉到其他頁面。

## 執行步骤

### 步骤1：頁面佈局規劃

為每個頁面設計佈局結構。

#### 1.2 定義頁面栅格系統

使用回應式栅格系統：

- **桌面端（>1200px）**：最大宽度1200px，居中顯示
- **平板端（768px-1200px）**：自適應宽度，内边距减小
- **移動端（<768px）**：單列佈局，表格格横向滚動

#### 1.3 確定關鍵元素的位置和大小

為每個頁面的關鍵元素規劃：

- 统計卡片：高度、列數、间距
- 篩選区：佈局方式（行内/垂直）、輸入框宽度
- 表格格：列宽、行高、最大高度
- 按钮：大小、间距、對齐方式
- 模態框：宽度、最大高度

### 步骤2：功能模組規劃

#### 2.1 统計卡片（如果需要）

**適用場景**：

- 仪表格板頁面
- 清單格頁顶部展示關鍵指標

**卡片內容**：

- 標題（如"待處理订單"）
- 數值（如"12"）
- 辅助說明（如"需要审批的订單"）
- 趋势指示器（可选，如"+12.5%"）

**設計要点**：

- 數量：通常2-6個
- 佈局：回應式网格，自動適應
- 樣式：卡片阴影、左侧色条、圖標等

#### 2.2 篩選搜尋区域

**常见篩選組件**：

- 文本搜尋框
- 下拉選擇器（單选/多选）
- 日期選擇器（單日/范围）
- 級联選擇器
- 快捷篩選標籤

**佈局方式**：

- **行内佈局**（推荐）：篩選條件在同一行，自動换行
- **垂直佈局**：篩選條件垂直排列，適合篩選項较多的場景

**操作按钮**：

- 搜尋/查詢按钮
- 清除/重設按钮
- 高級篩選展開按钮（可选）

#### 2.3 資料展示区

**表格格設計**：

```markdown
| 列名       | 宽度  | 對齐方式 | 說明     |
| ---------- | ----- | -------- | -------- |
| 選擇框     | 50px  | 居中     | 批量選擇 |
| 订單編號   | 120px | 左對齐   | 主键欄位 |
| 供應商名称 | 150px | 左對齐   | 文本欄位 |
| 订單金額   | 100px | 右對齐   | 數字欄位 |
| 狀態       | 100px | 居中     | 狀態標籤 |
| 操作       | 160px | 居中     | 操作按钮 |
```

**表格格特性**：

- 固定表格头（滚動時表格头不動）
- 行悬停高亮
- 斑马纹（可选）
- 空狀態提示

**卡片/清單格設計**（备选）：
適用於欄位较少或需要展示更多信息的場景。

#### 2.4 操作按钮区

**主要操作**（通常在篩選区下方）：

- 新建/新增按钮（主按钮樣式）
- 批量操作按钮（刪除、审批等）
- 匯入/匯出按钮
- 其他業務操作

**行内操作**（在表格格每行的操作列）：

- 檢視/詳情
- 編輯
- 刪除
- 其他業務操作（审批、提交等）

**按钮設計原則**：

- 主要操作使用主按钮樣式
- 危險操作（刪除）使用警告樣式
- 操作按钮不超过5個，多余的放入"更多"下拉菜單

#### 2.5 模態框（Modal）

**新增/編輯模態框**：

```
宽度：600px-900px（根据表格單複雜度）
結構：
- 头部：標題 + 關閉按钮
- 主体：表格單內容（分組展示）
- 底部：取消 + 確定按钮
```

**檢視詳情模態框**：

```
宽度：800px-1000px
結構：
- 头部：標題 + 關閉按钮
- 主体：詳情信息（分組展示）
- 底部：關閉 + 編輯按钮（可选）
```

**確認對话框**：

```
宽度：400px-500px
結構：
- 头部：警告標題
- 主体：確認提示內容
- 底部：取消 + 確定按钮
```

**匯入模態框**：

```
宽度：600px
結構：
- 檔案上傳区
- 模板下載連結
- 匯入结果展示区
- 失败结果下載連結
```

#### 2.6 分頁組件

**分頁信息**：

- 总条數
- 當前頁/总頁數
- 每頁顯示条數（可选）

**分頁操作**：

- 首頁/末頁
- 上一頁/下一頁
- 頁碼按钮（通常顯示5-7個頁碼）

### 步骤3：互動流程設計

#### 3.1 資料載入流程

```
頁面載入
  ↓
顯示載入動画
  ↓
載入模擬資料
  ↓
渲染頁面內容
  ↓
隐藏載入動画
```

#### 3.2 CRUD操作流程

**新增流程**：

```
点击"新增"按钮
  ↓
打开新增模態框
  ↓
產生自動欄位（如订單編號）
  ↓
使用者填写表格單
  ↓
点击"儲存"按钮
  ↓
表格單驗證
  ├─失败→ 顯示錯誤提示，停留在表格單
  └─成功
      ↓
    顯示載入狀態
      ↓
    模擬儲存資料
      ↓
    關閉模態框
      ↓
    重新整理清單格
      ↓
    顯示成功提示
```

**編輯流程**：

```
点击"編輯"按钮
  ↓
打开編輯模態框
  ↓
填充現有資料
  ↓
使用者修改表格單
  ↓
点击"儲存"按钮
  ↓
表格單驗證
  ├─失败→ 顯示錯誤提示
  └─成功
      ↓
    更新資料
      ↓
    關閉模態框
      ↓
    重新整理清單格
      ↓
    顯示成功提示
```

**刪除流程**：

```
点击"刪除"按钮
  ↓
顯示確認對话框
  ├─取消→ 關閉對话框
  └─確定
      ↓
    刪除資料
      ↓
    重新整理清單格
      ↓
    顯示成功提示
```

**檢視詳情流程**：

```
点击"檢視"按钮
  ↓
打开詳情模態框
  ↓
展示完整資料
  ↓
使用者檢視
  ↓
關閉模態框
```

#### 3.3 表格單驗證邏輯

**驗證時机**：

- 失焦驗證（欄位失去焦点時）
- 提交驗證（点击提交按钮時）

**驗證規則**：

```javascript
{
  order_no: {
    required: true,
    message: "订單編號不能為空"
  },
  supplier: {
    required: true,
    message: "请選擇供應商"
  },
  total_amount: {
    required: true,
    type: "number",
    min: 0,
    message: "订單金額必須大於0"
  },
  email: {
    pattern: /^[\w-]+(\.[\w-]+)*@[\w-]+(\.[\w-]+)+$/,
    message: "邮箱格式不正确"
  }
}
```

**錯誤顯示**：

- 輸入框边框變红
- 輸入框下方顯示錯誤提示文字
- 提交時滚動到第一個錯誤欄位

#### 3.4 狀態切换邏輯

如果有狀態管理，設計狀態流轉規則：

```
草稿(draft)
  ↓ [提交]
待审批(pending)
  ↓ [审批]
  ├→ 审批透過(approved)
  ├→ 审批拒绝(rejected)
  └→ 退回修改(returned) → 重新提交 → 待审批
```

**狀態標籤樣式**：

- 草稿：绿色
- 待审批：橙色
- 审批透過：绿色
- 审批拒绝：红色
- 退回修改：橙色

#### 3.5 錯誤處理和提示

**成功提示**：

```
類型：绿色訊息框
位置：頁面顶部居中
內容："操作成功！"
自動關閉：3秒
```

**錯誤提示**：

```
類型：红色訊息框
位置：頁面顶部居中
內容："操作失败：[具體原因]"
自動關閉：5秒或手動關閉
```

**警告提示**：

```
類型：橙色訊息框
內容："您確定要[操作]吗？此操作不可撤销。"
按钮：取消 / 確定
```
