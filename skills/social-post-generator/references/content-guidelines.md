# 內容撰寫準則與品質管理 (Content Guidelines)

本文檔提供社群發文內容撰寫的標準化準則，確保內容品質、合法性與組織形象。

---

## 一、核心價值準則

### 1.1 受害者中心 (Victim-Centered)

✅ **應該做到**：
- 以受害者及其家屬的權益為核心
- 尊重家屬的意願與隱私
- 賦權 (empower) 而非消費悲劇
- 突顯家屬的聲音與主體性

❌ **不應該**：
- 為了流量過度渲染血腥細節
- 未經同意使用家屬照片或語錄
- 將家屬塑造成「可憐」的客體
- 為了情緒效果誇大或捏造事實

### 1.2 正義導向 (Justice-Oriented)

✅ **應該做到**：
- 聚焦制度改善而非個人恩怨
- 理性陳述事實，避免情緒化攻擊
- 提出建設性解決方案
- 維持對話空間，願意溝通

❌ **不應該**：
- 人身攻擊法官、加害者或其他角色
- 散播仇恨言論
- 鼓勵非法行動
- 製造對立而非尋求共識

### 1.3 真實透明 (Authentic & Transparent)

✅ **應該做到**：
- 所有數據、日期、引用均有來源
- 區分事實陳述與意見表達
- 錯誤發生時主動更正
- 清楚標示資訊來源

❌ **不應該**：
- 使用未經查證的傳言或假新聞
- 斷章取義扭曲原意
- 隱瞞或淡化重要資訊
- 冒充他人發言

## 二、法律與倫理紅線

### 2.1 個資保護

**個資法規範**：
- 未經同意不得公開家屬個人資料（姓名、地址、聯絡方式等）
- 已公開的司法資訊（如新聞報導中的家屬姓名）可使用，但需謹慎
- 特殊身份者（如未成年人、性侵害被害人）需額外保護

**實務準則**：
```
✅ 可以：「受害者母親表示...」（新聞已公開）
❌ 不可：「王小姐住在台北市...」（未公開的個資）
✅ 可以：「家屬在記者會上...」（公開場合）
❌ 不可：「家屬私下告訴我...」（未公開的私下談話）
```

### 2.2 誹謗風險

**高風險用詞避免**：
- ❌ 「貪污法官」、「黑箱司法」（無具體證據）
- ❌ 「關說」、「利益輸送」（未經證實）
- ❌ 「包庇」、「縱容」（涉及意圖難證明）

**安全替代用詞**：
- ✅ 「輕判引發爭議」（陳述事實）
- ✅ 「量刑考量引發討論」（客觀描述）
- ✅ 「制度設計可能存在漏洞」（聚焦制度）

### 2.3 著作權與肖像權

**圖片使用**：
- ✅ 使用已授權的新聞圖片（標示來源）
- ✅ 使用組織自製圖片
- ✅ 使用創用CC授權圖片（遵守授權條款）
- ❌ 直接下載使用媒體照片（可能侵權）
- ❌ 使用私人Facebook照片（未經授權）

**文字引用**：
- 新聞報導摘要：合理使用範圍內
- 專家語錄：標示姓名與來源
- 書籍/論文：標示出處

## 三、內容品質標準

### 3.1 事實查核清單

**發布前必檢查**：

- [ ] **人名正確性**：所有姓名拼寫正確
- [ ] **日期準確性**：事件時間、判決日期正確
- [ ] **數據來源**：統計數據有明確來源
- [ ] **法律條文**：引用的法條為現行有效版本
- [ ] **機構名稱**：機關、組織全名正確
- [ ] **職稱正確**：法官、檢察官、立委等職稱正確

### 3.2 情緒尺度管理

**情感訴求的界線**：

✅ **適度**：
- 陳述家屬真實感受
- 描繪復原歷程的希望
- 引發同理心與支持

❌ **過度**：
- 詳細描述血腥或殘忍細節
- 反覆刺激讀者的恐懼或悲傷
- 使用煽情標題騙取點擊

**檢測問題**：
- 如果我是家屬，我會願意這樣被描述嗎？
- 這個內容是為了支持家屬，還是為了組織曝光？
- 去掉情緒化用詞後，事實還成立嗎？

### 3.3 平衡陳述原則

**即使持有立場，也應該**：
- 準確陳述對方觀點（再進行回應）
- 承認我方資訊的侷限
- 區分事實、推測、意見
- 承認不確定性（「目前資訊顯示...」）

## 四、平台特定規範

### 4.1 Facebook

**特性**：
- 字數寛鬆，可深度論述
- 適合連結分享與討論
- 留言區可能出現負面聲音

**準則**：
- 長文可用「...查看更多」折疊
- 主動提供參考連結供查證
- 預設回應範本應對常見質疑

### 4.2 Instagram

**特性**：
- 視覺導向，圖片比文字重要
- 前2行決定是否展開全文
- 限時動態適合即時性內容

**準則**：
- 開頭2行必須抓住眼球
- 使用表情符號增加可讀性
- Hashtag可放在第一則留言

### 4.3 Threads

**特性**：
- 輕鬆、對話式氛圍
- 字數限制500字
- 適合即時互動與串文

**準則**：
- 語氣可較口語化
- 主動邀請回應與討論
- 即時回覆留言建立連結

## 五、發布前檢查清單

### 5.1 基礎檢查（必做）

**內容正確性**：
- [ ] 所有事實已查核
- [ ] 日期、數據正確
- [ ] 人名、機構名正確
- [ ] 連結可正常開啟

**法律合規性**：
- [ ] 無誹謗或攻擊性言論
- [ ] 無洩漏未公開個資
- [ ] 圖片已取得授權或標示來源
- [ ] 引用內容標示出處

**品質完整性**：
- [ ] 無明顯錯字或語法錯誤
- [ ] 標點符號正確
- [ ] 格式一致（分段、換行）
- [ ] Hashtag無誤拼

### 5.2 進階檢查（建議）

**策略適配性**：
- [ ] 符合當日社群經營主題
- [ ] 與其他貼文時間錯開
- [ ] 非敏感時段（如重大災難發生時）
- [ ] 已考慮可能的輿論反應

**家屬權益**：
- [ ] 使用家屬照片已取得授權
- [ ] 引用家屬語錄準確且完整
- [ ] 未過度消費家屬悲傷
- [ ] 家屬形象正面且獲得尊重

## 六、危機處理準則

### 6.1 發現內容錯誤

**處理流程**：
1. 立即下架或編輯錯誤貼文
2. 發布更正聲明（標示更正處）
3. 私訊或標記已受誤導的用戶
4. 記錄錯誤原因，避免再次發生

**更正聲明範本**：
```
【更正聲明】

稍早發布的貼文中，[具體錯誤內容]有誤，正確資訊應為[正確內容]。

我們已更正原貼文，並感謝 [指出錯誤者] 的提醒。

我們將更加嚴謹查核資訊，避免類似情況再次發生。
```

### 6.2 面對負面回應

**區分類型**：
- **善意質疑**：提供資訊、邀請對話
- **惡意攻擊**：封鎖/刪除，不回應
- **錯誤資訊**：溫和更正，提供證據

**不回應原則**：
- 人身攻擊不回應
- 明顯挑釁不回應
- 無理取鬧不回應

**回應範本（善意質疑）**：
```
感謝您的提問。關於[問題]，我們的資訊來自[來源]。

如果您需要更多背景資料，可以參考[連結]。

我們也歡迎進一步的討論。
```
